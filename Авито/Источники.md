# Источники Avito

## Инвентаризация

В исходной папке было 61 файл: 3 TXT, 1 HTML, 2 PDF, 53 PNG и 2 служебных `.DS_Store`. В vault скопированы 59 содержательных файлов общим объёмом 96 644 133 bytes (92,17 MiB); `.DS_Store` исключены. Документы лежат в `90 Вложения/Авито`, изображения — в `90 Вложения/Авито/Скриншоты`.

После сопоставления файлов по имени проверка размера и SHA-256 подтвердила совпадение всех 59 копий с исходной папкой на 2026-07-18.

Точных побайтовых дублей среди содержательных файлов нет. При этом несколько файлов повторяют одни и те же условия в другом формате или с немного иной полнотой.

## Неоднозначности и устаревшие формулировки

### Loop variables в Go

Исходное объяснение задачи о goroutine в цикле описывает старую семантику capture. Для переменных, объявленных самим `for`/`range`, module language version начиная с Go 1.22 создаёт отдельную переменную на итерацию. Это не отменяет необходимость ждать goroutines и синхронизировать общий `max`. Граница подтверждается [Go 1.22 release notes](https://go.dev/doc/go1.22) и [Go specification](https://go.dev/ref/spec#For_statements).

### Алгоритмические пропуски

- В «Сравнении версий» источник задаёт необычный знак: `1`, если левая версия меньше правой, и `-1`, если больше. Решение сохраняет этот контракт, а не подменяет его соглашением стандартных comparator-функций.
- В дереве рубрикатора ожидаемый вывод содержит `Вещи => Одежда => Женская`, хотя узел `Женская` отсутствует во входной схеме. Решение показывает и исходное противоречие, и минимальную нормализацию дерева.
- Для «Пора в отпуск» TXT потерял числовой ответ. Скриншот восстанавливает `[5, 0]` для полного примера; `[10, 6]` сохранился без полного входа и не используется как исполняемый test case.
- Часть примеров в Go-задачах обрезана. Решения отделяют наблюдаемое условие от допущений и не восстанавливают неизвестные требования.

### Теоретические упрощения

- Ожидаемая стоимость map lookup не даёт worst-case гарантии `O(1)`.
- У встроенного Go map нет пользовательского `cap` с контрактом, аналогичным slice.
- HTTPS не означает, что весь трафик шифруется асимметричным алгоритмом: TLS использует handshake для согласования и аутентификации, затем симметричную защиту записей.
- Ответ про выделение 1 КБ памяти зависит от allocator, runtime, страниц ОС, GC и лимитов процесса; один системный вызов на каждую allocation не предполагается.
- `context.Context` переносит deadline, cancellation и request-scoped values по дереву вызовов, но не заменяет универсальное «управление потоком».

## Правило доказательности

Исходники Avito подтверждают, что задача или вопрос встречались в наборе, но не служат первичным источником по поведению Go, PostgreSQL, Kubernetes или Linux. Версионные и системные утверждения в решениях сверяются с официальной спецификацией, документацией, release notes или исходным кодом фиксированной версии.
