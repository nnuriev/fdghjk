# Указатели

## TL;DR

Go передаёт аргументы по значению. Параметр `person *Person` — копия pointer value caller. Присваивание нового адреса этой копии не меняет pointer caller, поэтому исходная программа печатает `Bob`, затем снова `Bob`. Чтобы получить `Bob`, затем `Alice`, достаточно изменить объект через `person.Name = "Alice"`.

## Условие и контракт

Исходная функция получает `*Person`, присваивает параметру адрес нового `Person{Name: "Alice"}`, а caller печатает имя до и после вызова. Нужно объяснить вывод и показать способы изменить наблюдаемое состояние.

Контракт решения:

- `Rebind` демонстрирует исходное поведение и не меняет caller;
- `Mutate` меняет поле существующего объекта, сохраняя identity;
- `Replace` принимает `**Person` и меняет сам pointer caller;
- все функции требуют ненулевые pointers.

### Неоднозначности исходника

- Требование «чтобы вывелось Bob/Alice» не уточняет, должна ли сохраниться identity объекта. `Mutate` сохраняет, `Replace` — нет.
- В Go нет отдельной семантики pass-by-reference для параметров. Поведение `*Person` следует из обычного копирования pointer value.
- Обработка `nil` не задана. В минимальном интервью-контракте `nil` считается нарушением предусловия; production API может вернуть ошибку вместо panic.

## Ментальная модель

Pointer — значение, содержащее адрес variable. После вызова `Rebind(person)` есть два pointer values с одинаковым адресом: один у caller, другой в параметре. Строка `person = &Person{...}` меняет только локальный pointer. Оба pointer values не становятся «ссылкой на слот caller».

Операция `person.Name = "Alice"` идёт по адресу и меняет общий объект. `*Person` позволяет изменить pointee, а `**Person` — слот, в котором хранится pointer. Эта граница дополняет общую value semantics из [[60 Go/Zero values, семантика значений и копирование|заметки о копировании значений]] и правила адресуемости из [[60 Go/Структуры, указатели и методы|заметки о структурах и указателях]].

## Concurrency invariants

В задаче concurrency нет. Pointer сам ничего не синхронизирует: если несколько goroutines одновременно читают и меняют `Person.Name`, потребуется happens-before через mutex, channel или другой механизм. Копирование адреса не делает pointee immutable и не защищает его от data race.

## Код Go 1.26.5

`pointers.go`:

```go
package pointers

type Person struct {
	Name string
}

// Rebind changes only the local copy of the pointer.
func Rebind(person *Person) {
	person = &Person{Name: "Alice"}
}

// Mutate changes the object to which both pointer copies refer.
func Mutate(person *Person) {
	person.Name = "Alice"
}

// Replace changes the caller's pointer itself.
func Replace(person **Person) {
	*person = &Person{Name: "Alice"}
}
```

## Tests

`pointers_test.go`:

```go
package pointers

import "testing"

func TestRebindDoesNotChangeCallerPointer(t *testing.T) {
	person := &Person{Name: "Bob"}
	original := person

	Rebind(person)

	if person != original || person.Name != "Bob" {
		t.Fatalf("Rebind changed caller state: person=%p name=%q", person, person.Name)
	}
}

func TestMutateAndReplace(t *testing.T) {
	person := &Person{Name: "Bob"}
	original := person

	Mutate(person)
	if person != original || person.Name != "Alice" {
		t.Fatalf("Mutate() person=%p name=%q, want same pointer and Alice", person, person.Name)
	}

	person.Name = "Bob"
	Replace(&person)
	if person == original || person.Name != "Alice" {
		t.Fatalf("Replace() person=%p name=%q, want new pointer and Alice", person, person.Name)
	}
}
```

## Проверка результата

Для исходного варианта `Rebind` имя остаётся `Bob`. `Mutate` даёт `Alice` с тем же адресом объекта. `Replace` также даёт `Alice`, но pointer начинает указывать на новый объект. Код и tests прошли `go test`, `go vet` и `go test -race` на Go 1.26.5 `darwin/arm64`, проверено 2026-07-18.

## Сложность и ресурсы

Все операции занимают `O(1)` времени. `Rebind` и `Replace` создают новый `Person`; фактическое размещение определяет escape analysis. `Mutate` не создаёт новый объект.

## Trade-offs и альтернативы

- Mutation через `*Person` сохраняет identity и aliases, но делает side effect частью контракта.
- `**Person` технически меняет pointer caller, однако API сложнее читать. Часто яснее вернуть `*Person` и присвоить результат явно: `person = RenameCopy(person)`.
- Передача `Person` по значению и возврат новой копии уменьшает aliasing. Цена зависит от размера структуры и требуемой identity; автоматически заменять pointer на value не следует.

## Типичные ошибки

- **Предположение:** `*Person` передаётся by reference. **Симптом:** caller продолжает видеть старый адрес после `person = &Person{...}`. **Причина:** скопирован pointer value. **Исправление:** изменить pointee, вернуть новый pointer или принять `**Person`.
- **Предположение:** любое изменение через локальный pointer невидимо caller. **Симптом:** поле неожиданно меняется. **Причина:** обе копии pointer указывают на один объект. **Исправление:** различать rebinding pointer и mutation pointee.
- **Предположение:** pointer автоматически решает concurrency. **Симптом:** race detector на поле `Name`. **Причина:** общий адрес создаёт aliasing, а не happens-before. **Исправление:** определить владельца и synchronization policy.

## Когда применять выводы

Та же модель нужна для параметров `*T`, элементов linked structures, optional mutable state и API, возвращающих pointers. На собеседовании сначала нарисуйте два pointer slots и один объект: это быстрее показывает, какая именно storage location меняется.
