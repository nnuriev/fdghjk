# Удаление нулей из массива inplace

## TL;DR

Указатель `write` отмечает позицию следующего ненулевого элемента. `read` сканирует весь слайс и стабильно уплотняет ненули в начало; затем хвост от `write` заполняется нулями. Получаем `O(N)` времени и `O(1)` дополнительной памяти.

## Нормализованное условие и контракт

Функция изменяет переданный `[]int` на месте: все ненулевые элементы переходят в начало в прежнем относительном порядке, оставшиеся позиции заполняются нулями. Длина и backing array не меняются, отдельный результат не создаётся.

### Неоднозначности исходника

Название говорит «удалить нули», но пример сохраняет длину и сдвигает нули в конец. Поэтому точная операция — stable partition `non-zero | zero`, а не сокращение слайса.

`inplace` здесь означает `O(1)` дополнительной памяти и mutation элементов. Go slice descriptor передаётся по значению, но он указывает на общий backing array, поэтому вызывающая сторона видит изменения элементов.

## Ментальная модель

Слайс делится на три зоны:

```text
[0:write)       — уже уплотнённые ненули
[write:read)    — обработанная зона, её старые значения больше не важны
[read:len)      — ещё не просмотренный хвост
```

Каждый новый неноль расширяет первую зону на один элемент.

## Алгоритм и корректность

Для каждого `value` слева направо, если он ненулевой, записываем его в `nums[write]` и увеличиваем `write`. После сканирования заполняем `nums[write:]` нулями.

Инвариант перед чтением `read`: `nums[:write]` равен последовательности всех ненулевых элементов исходного префикса `nums[:read]` в исходном порядке. Ноль не меняет эту последовательность; неноль дописывается в её конец, поэтому инвариант сохраняется. После полного прохода префикс содержит все ненули, а заполнение хвоста создаёт нужное число нулей, не затрагивая порядок префикса.

## Код Go 1.26.5

`solution.go`:

```go
package movezeroes

func MoveZeroes(nums []int) {
	write := 0
	for _, value := range nums {
		if value != 0 {
			nums[write] = value
			write++
		}
	}
	for write < len(nums) {
		nums[write] = 0
		write++
	}
}
```

`solution_test.go`:

```go
package movezeroes

import (
	"reflect"
	"testing"
)

func TestMoveZeroes(t *testing.T) {
	tests := []struct {
		name  string
		input []int
		want  []int
	}{
		{
			name:  "source example",
			input: []int{7, 3, 0, 0, 0, 2, 4, 0, 5, 19},
			want:  []int{7, 3, 2, 4, 5, 19, 0, 0, 0, 0},
		},
		{name: "already compact", input: []int{1, 2, 3}, want: []int{1, 2, 3}},
		{name: "all zeroes", input: []int{0, 0}, want: []int{0, 0}},
		{name: "stability", input: []int{0, -2, 0, 1, -3}, want: []int{-2, 1, -3, 0, 0}},
		{name: "empty", input: nil, want: nil},
	}

	for _, tc := range tests {
		t.Run(tc.name, func(t *testing.T) {
			MoveZeroes(tc.input)
			if !reflect.DeepEqual(tc.input, tc.want) {
				t.Fatalf("got %v, want %v", tc.input, tc.want)
			}
		})
	}
}
```

## Сложность

Каждый элемент читается один раз; не более `N` позиций записываются при уплотнении и не более `N` — при очистке хвоста. Время `O(N)`, дополнительная память `O(1)`.

## Trade-offs

Swap ненуля с позицией `write` объединяет работу в один цикл и тоже стабилен для ненулей, но делает лишние записи нулей в середину и сложнее объясняется через две фазы. Возврат `nums[:write]` действительно «удалил» бы нули из логической длины, но не удовлетворяет примеру, который требует нули в хвосте исходной длины.

## Типичные ошибки

- **Вернуть новый слайс ненулей → нарушено `O(1)` и mutation contract → создана дополнительная память → писать в исходный backing array.**
- **Менять неноль с последним элементом → порядок ненулей ломается → используется unstable partition → уплотнять слева направо.**
- **Не занулить хвост → старые значения дублируются после compact prefix → запись ненулей не очищает остаток → выполнить вторую фазу.**
