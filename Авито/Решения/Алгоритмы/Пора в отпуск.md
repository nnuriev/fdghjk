# Пора в отпуск

## TL;DR

Преобразуем разреженный список встреч в массив по дням и считаем сумму первого окна длины `V`. Затем сдвигаем окно на один день: вычитаем ушедший день и добавляем вошедший. Сохраняем только строго меньшую сумму, поэтому при равенстве возвращается самый ранний старт.

## Нормализованное условие и контракт

Завтра — день `1`. Весь отпуск длины `vacationLength=V` должен лежать внутри дней `1..periodLength=P`, значит допустимые старты — `1..P-V+1`. Встреча, назначенная на любой день отпуска, считается пропущенной.

Вход содержит неотрицательное число встреч для некоторых уникальных дней. Неупомянутые дни имеют ноль встреч. Нужно вернуть пару `(самый ранний день старта среди оптимальных, минимальное число пропущенных встреч)`. Требуется `P>0`, `V>0`, `V<=P`; дни встреч должны лежать в `1..P`, дубликаты дней и переполнение суммы — ошибки.

### Зафиксированные ответы и повреждения источника

Для полного примера

```text
meetings = [(3,1), (4,3), (14,3), (21,3), (28,1)]
P = 30, V = 7
```

допустимы старты `1..24`; самый ранний нулевой интервал — дни `5..11`, поэтому ответ **`[5,0]`**.

В одном из приложенных фрагментов также сохранился ожидаемый ответ **`[10,6]`**, но вход этого примера отсутствует. Эта пара зафиксирована как часть источника, однако не используется как test oracle: без календаря невозможно проверить ни оптимальность, ни допустимость старта. Восстанавливать вход по ответу означало бы выдумать условие.

## Ментальная модель

Два соседних отпуска длины `V` отличаются ровно двумя днями. Поэтому не нужно пересчитывать все `V` значений: состояние окна обновляется дельтой `-left + newRight`.

## Алгоритм и корректность

1. Валидируем события и записываем число встреч в `byDay[day]`.
2. Суммируем дни `1..V`; это окно со стартом `1`.
3. Для каждого следующего старта вычитаем предыдущий левый день и добавляем новый правый.
4. Обновляем ответ только при `window < best`.

Инвариант перед сравнением старта `s`: `window` равен сумме `byDay[s:s+V]`. Он верен для `s=1` по построению. Переход удаляет единственный день старого окна вне нового и добавляет единственный новый день, поэтому сохраняет инвариант. Перебираются все допустимые старты; строгий update сохраняет первый старт при равенстве.

## Код Go 1.26.5

`solution.go`:

```go
package vacation

import (
	"errors"
	"math"
)

type MeetingDay struct {
	Day      int
	Meetings int64
}

type Choice struct {
	StartDay       int
	MissedMeetings int64
}

func BestVacation(days []MeetingDay, periodLength, vacationLength int) (Choice, error) {
	if periodLength <= 0 || vacationLength <= 0 || vacationLength > periodLength {
		return Choice{}, errors.New("require 0 < vacationLength <= periodLength")
	}

	byDay := make([]int64, periodLength+1)
	seen := make([]bool, periodLength+1)
	for _, item := range days {
		if item.Day < 1 || item.Day > periodLength {
			return Choice{}, errors.New("meeting day is outside the period")
		}
		if item.Meetings < 0 {
			return Choice{}, errors.New("meeting count must be non-negative")
		}
		if seen[item.Day] {
			return Choice{}, errors.New("duplicate meeting day")
		}
		seen[item.Day] = true
		byDay[item.Day] = item.Meetings
	}

	var window int64
	for day := 1; day <= vacationLength; day++ {
		if math.MaxInt64-window < byDay[day] {
			return Choice{}, errors.New("meeting sum overflows int64")
		}
		window += byDay[day]
	}
	best := Choice{StartDay: 1, MissedMeetings: window}

	lastStart := periodLength - vacationLength + 1
	for start := 2; start <= lastStart; start++ {
		window -= byDay[start-1]
		entering := byDay[start+vacationLength-1]
		if math.MaxInt64-window < entering {
			return Choice{}, errors.New("meeting sum overflows int64")
		}
		window += entering
		if window < best.MissedMeetings {
			best = Choice{StartDay: start, MissedMeetings: window}
		}
	}
	return best, nil
}
```

package vacation

import "testing"

func TestBestVacation(t *testing.T) {
	tests := []struct {
		name        string
		days        []MeetingDay
		period, vac int
		want        Choice
		wantErr     bool
	}{
		{
			name:   "complete source example",
			days:   []MeetingDay{{3, 1}, {4, 3}, {14, 3}, {21, 3}, {28, 1}},
			period: 30, vac: 7,
			want: Choice{StartDay: 5, MissedMeetings: 0},
		},
		{
			name:   "minimum at first window",
			days:   []MeetingDay{{1, 2}, {2, 1}, {3, 5}},
			period: 3, vac: 2,
			want: Choice{StartDay: 1, MissedMeetings: 3},
		},
		{name: "tie chooses earliest", days: nil, period: 5, vac: 2, want: Choice{StartDay: 1}},
		{name: "vacation longer than period", period: 2, vac: 3, wantErr: true},
		{
			name: "duplicate day", days: []MeetingDay{{1, 1}, {1, 2}},
			period: 3, vac: 1, wantErr: true,
		},
	}

	for _, tc := range tests {
		t.Run(tc.name, func(t *testing.T) {
			got, err := BestVacation(tc.days, tc.period, tc.vac)
			if (err != nil) != tc.wantErr {
				t.Fatalf("error = %v, wantErr %v", err, tc.wantErr)
			}
			if !tc.wantErr && got != tc.want {
				t.Fatalf("got %+v, want %+v", got, tc.want)
			}
		})
	}
}

## Сложность

При `M` записях встреч время `O(P+M)`, память `O(P)`. Если дни уже отсортированы, можно обновлять окно по sparse events и снизить память до `O(M)`, но реализация и обработка границ становятся сложнее.

## Trade-offs

Prefix sums также дают `O(P+M)` подготовки и `O(1)` на окно, но хранят сумму каждого префикса; sliding window хранит только текущую сумму. Prefix sums выигрывают, если после подготовки поступает много запросов с разными интервалами. Полный пересчёт каждого отпуска стоит `O(P·V)`.

## Типичные ошибки

- **Перебирать старты до `P` → часть отпуска выходит за горизонт → перепутаны начало и полное размещение → последний старт `P-V+1`.**
- **Обновлять ответ при `<=` → выбирается последний оптимальный старт → tie-break не зафиксирован → использовать `<` для самого раннего.**
- **Трактовать неупомянутый день как ошибку → разреженный календарь становится непригоден → отсутствие записи означает ноль встреч → инициализировать массив нулями.**
- **Превратить `[10,6]` без входа в тест → тест подтверждает выдуманный календарь → oracle неполон → сохранить артефакт, но не выдавать его за проверенный пример.**
