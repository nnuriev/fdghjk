# Сумма чисел двух массивов

## TL;DR

Идём справа налево, складываем две цифры и перенос. В текущую позицию результата пишем `sum % 10`, переносом становится `sum / 10`. Буфер длины `max(len(a),len(b))+1` заранее покрывает возможный новый старший разряд.

## Нормализованное условие и контракт

Каждый непустой слайс хранит десятичное неотрицательное целое: один элемент — одна цифра `0..9`, старшая цифра идёт первой. Нужно вернуть сумму в том же формате. Вход может иметь ведущие нули, результат канонизируется: ведущих нулей нет, кроме единственного `[0]` для числа ноль. Входы не изменяются.

### Неоднозначности исходника

- Фраза «массив неотрицательных чисел» шире примеров. Для однозначного десятичного представления элементы нормализованы именно как цифры `0..9`.
- Исходник допускает один незначащий ноль в ответе. Здесь выбран более строгий канонический результат.
- Пустой массив не определяет число; вместо неявного нуля функция возвращает ошибку.

## Ментальная модель

Это письменное сложение, только колонка адресуется индексами. После обработки суффиксов перенос — единственная информация, которая связывает уже посчитанную младшую часть с ещё не обработанной старшей.

## Алгоритм и корректность

Пусть указатели `i`, `j` идут от концов входов, а `write` — от конца результата. На каждом шаге складываем существующие цифры и `carry`, записываем младшую цифру суммы и переносим десятки.

Инвариант: после заполнения позиций правее `write` они точно представляют соответствующий суффикс суммы, а `carry` равен переносу в следующую старшую колонку. Правила `% 10` и `/ 10` сохраняют инвариант. После обработки всех колонок позиция `0` содержит финальный перенос либо ноль; удаление ведущих нулей не меняет числового значения.

## Код Go 1.26.5

`solution.go`:

```go
package digitssum

import "errors"

func AddDigits(a, b []int) ([]int, error) {
	if len(a) == 0 || len(b) == 0 {
		return nil, errors.New("both numbers must contain at least one digit")
	}
	if !validDigits(a) || !validDigits(b) {
		return nil, errors.New("every element must be a decimal digit")
	}

	maxLen := len(a)
	if len(b) > maxLen {
		maxLen = len(b)
	}
	result := make([]int, maxLen+1)
	i, j := len(a)-1, len(b)-1
	carry := 0
	for write := maxLen; write >= 0; write-- {
		sum := carry
		if i >= 0 {
			sum += a[i]
			i--
		}
		if j >= 0 {
			sum += b[j]
			j--
		}
		result[write] = sum % 10
		carry = sum / 10
	}

	first := 0
	for first < len(result)-1 && result[first] == 0 {
		first++
	}
	return result[first:], nil
}

func validDigits(digits []int) bool {
	for _, digit := range digits {
		if digit < 0 || digit > 9 {
			return false
		}
	}
	return true
}
```

## Table-driven tests

`solution_test.go`:

```go
package digitssum

import (
	"reflect"
	"testing"
)

func TestAddDigits(t *testing.T) {
	tests := []struct {
		name    string
		a, b    []int
		want    []int
		wantErr bool
	}{
		{name: "without carry", a: []int{1, 2, 3}, b: []int{4, 5, 6}, want: []int{5, 7, 9}},
		{name: "new most significant digit", a: []int{5, 4, 4}, b: []int{4, 5, 6}, want: []int{1, 0, 0, 0}},
		{name: "different lengths", a: []int{9, 9}, b: []int{1}, want: []int{1, 0, 0}},
		{name: "canonical zero", a: []int{0, 0}, b: []int{0}, want: []int{0}},
		{name: "leading input zeros", a: []int{0, 0, 1}, b: []int{9}, want: []int{1, 0}},
		{name: "invalid digit", a: []int{10}, b: []int{1}, wantErr: true},
		{name: "empty number", a: nil, b: []int{1}, wantErr: true},
	}

	for _, tc := range tests {
		t.Run(tc.name, func(t *testing.T) {
			got, err := AddDigits(tc.a, tc.b)
			if (err != nil) != tc.wantErr {
				t.Fatalf("error = %v, wantErr %v", err, tc.wantErr)
			}
			if !tc.wantErr && !reflect.DeepEqual(got, tc.want) {
				t.Fatalf("got %v, want %v", got, tc.want)
			}
		})
	}
}
```

## Сложность

Время `O(max(len(a),len(b)))`, возвращаемая память `O(max(len(a),len(b))+1)`, дополнительное состояние помимо результата — `O(1)`.

## Trade-offs

Преобразование в `math/big.Int` снимает ручную арифметику, но требует сначала сериализовать представление и скрывает центральный инвариант задачи. Запись результата через `append` в обратном порядке также линейна, однако затем нужен reverse; заранее выделенный буфер сразу формирует правильный порядок.

## Типичные ошибки

- **Забыть финальный перенос → `544+456` даёт `000` → буфер не имеет старшего разряда → выделить `maxLen+1`.**
- **Идти слева направо → перенос приходит в уже записанную позицию → десятичное сложение зависит от младших колонок → идти с конца.**
- **Разрешить элементы больше `9` → representation перестаёт быть позиционной записью по одной цифре → контракт размыт → валидировать каждую цифру.**
- **Удалить все нули результата → сумма нулей становится пустым массивом → канонизация потеряла число → оставить один последний ноль.**
