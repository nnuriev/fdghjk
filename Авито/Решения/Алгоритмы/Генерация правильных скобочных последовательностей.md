# Генерация правильных скобочных последовательностей

## TL;DR

Строим строку слева направо. Открывающую скобку можно добавить, пока их меньше `N`; закрывающую — только когда уже открытых больше, чем закрытых. Так backtracking никогда не создаёт префикс, который невозможно завершить до правильной последовательности.

## Нормализованное условие и контракт

Для целого `N >= 0` нужно вернуть все правильные скобочные последовательности (ПСП) ровно из `N` пар одного вида `()`. Порядок детерминирован: ветка `(` обходится раньше ветки `)`. Для `N=0` результат — список из одной пустой строки: это единственная корректная последовательность из нуля пар. Отрицательное `N` — ошибка.

### Неоднозначности исходника

- Поведение при `N=0` не задано. Выбран математически стандартный базовый случай `C₀=1`, полезный и для рекурсии.
- Не указан предел `N`. Количество результатов растёт как число Каталана, поэтому вызывающая сторона должна ограничивать `N` доступной памятью; это не задача с компактным выводом.
- Порядок результатов в исходнике совпадает с DFS, где сначала добавляется `(`, но явно не объявлен обязательным. Здесь он включён в контракт.

## Ментальная модель

Правильность можно проверять по каждому префиксу: баланс `open-close` никогда не отрицателен, а в конце равен нулю. Backtracking поддерживает этот инвариант конструктивно, поэтому фильтровать `2^(2N)` произвольных строк после генерации не нужно.

## Алгоритм и корректность

Состояние рекурсии — число использованных открывающих и закрывающих скобок и общий буфер.

- Добавляем `(`, если `open < N`.
- Добавляем `)`, если `close < open`.
- При длине `2N` копируем буфер в ответ.

Любой сгенерированный префикс имеет `0 <= close <= open <= N`; в листе использовано `N` открывающих, длина равна `2N`, значит закрывающих тоже `N` и строка корректна. Обратно, у любой ПСП следующий символ в каждом префиксе удовлетворяет одному из этих двух правил, поэтому DFS повторит её путь. Разные пути отличаются первой выбранной скобкой и не дают дубликатов.

## Код Go 1.26.5

`solution.go`:

```go
package parentheses

import "errors"

func GenerateParentheses(n int) ([]string, error) {
	if n < 0 {
		return nil, errors.New("n must be non-negative")
	}

	result := make([]string, 0)
	buffer := make([]byte, 0, 2*n)
	var generate func(open, close int)
	generate = func(open, close int) {
		if len(buffer) == 2*n {
			result = append(result, string(buffer))
			return
		}
		if open < n {
			buffer = append(buffer, '(')
			generate(open+1, close)
			buffer = buffer[:len(buffer)-1]
		}
		if close < open {
			buffer = append(buffer, ')')
			generate(open, close+1)
			buffer = buffer[:len(buffer)-1]
		}
	}

	generate(0, 0)
	return result, nil
}
```

## Table-driven tests

`solution_test.go`:

```go
package parentheses

import (
	"reflect"
	"testing"
)

func TestGenerateParentheses(t *testing.T) {
	tests := []struct {
		name    string
		n       int
		want    []string
		wantErr bool
	}{
		{name: "zero pairs", n: 0, want: []string{""}},
		{name: "one pair", n: 1, want: []string{"()"}},
		{
			name: "three pairs", n: 3,
			want: []string{"((()))", "(()())", "(())()", "()(())", "()()()"},
		},
		{name: "negative", n: -1, wantErr: true},
	}

	for _, tc := range tests {
		t.Run(tc.name, func(t *testing.T) {
			got, err := GenerateParentheses(tc.n)
			if (err != nil) != tc.wantErr {
				t.Fatalf("error = %v, wantErr %v", err, tc.wantErr)
			}
			if !tc.wantErr && !reflect.DeepEqual(got, tc.want) {
				t.Fatalf("got %q, want %q", got, tc.want)
			}
		})
	}
}
```

## Сложность

Число результатов — `C_N = (1/(N+1))·binom(2N,N)`. На материализацию каждой строки нужно `O(N)`, поэтому время и память ответа — `Θ(N·C_N)`. Рабочий буфер и глубина рекурсии занимают `O(N)` сверх результата.

## Trade-offs

Генерация всех битовых строк длины `2N` с последующей проверкой требует `Θ(4^N·N)` работы и тратит время на заведомо плохие префиксы. Итеративный стек устраняет рекурсивные вызовы, но хранит те же состояния и обычно сложнее читается. Для lazy-потребителя можно выдавать строки callback-ом и не хранить весь ответ, однако количество вычислений не меняется.

## Типичные ошибки

- **Разрешать `)` при `close == open` → появляется отрицательный баланс префикса → такая строка уже неисправима → условие должно быть строгим `close < open`.**
- **Добавлять строку в ответ без копирования буфера в языках с mutable string buffer → все ответы указывают на одно состояние → нарушено владение → в Go conversion `string(buffer)` создаёт неизменяемое значение.**
- **Считать `N=0` отсутствием результатов → ломается базовый комбинаторный случай → пустая строка считается одной ПСП.**
