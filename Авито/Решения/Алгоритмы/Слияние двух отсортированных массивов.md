# Слияние двух отсортированных массивов

## TL;DR

Два указателя стоят на первых ещё не перенесённых элементах входов. На каждом шаге переносим меньший элемент, а при равенстве — элемент первого массива. Когда один вход закончился, дописываем остаток второго.

## Нормализованное условие и контракт

Даны два слайса `a` и `b`, каждый отсортирован неубывающе. Нужно вернуть новый неубывающий слайс длины `len(a)+len(b)`, содержащий все элементы обоих входов с сохранением кратности. Входы не изменяются. Нарушение сортировки считается ошибкой.

### Неоднозначности исходника

- Формулировка говорит «по возрастанию», но примеры содержат равные элементы. Поэтому точное предусловие — неубывающий порядок.
- Не задана политика для некорректного входа. Здесь она проверяется явно, а не приводит к тихо несортированному результату.
- Для равных значений происхождение в `[]int` ненаблюдаемо. Выбор первого массива делает merge стабильным, если элементы позднее обобщить до записей с ключом.

## Ментальная модель

Необработанные хвосты обоих массивов уже отсортированы. Их глобальный минимум обязательно находится в одной из двух голов. После переноса минимума утверждение остаётся верным для укороченного хвоста.

## Алгоритм и корректность

Поддерживаем индексы `i`, `j` и результат `out`. Инвариант перед каждой итерацией:

- `out` отсортирован;
- `out` содержит ровно элементы `a[:i]` и `b[:j]`;
- каждый элемент `out` не больше любого элемента необработанных хвостов.

Перенос меньшей головы сохраняет все три свойства. Когда один хвост пуст, второй уже отсортирован и каждый его элемент не меньше последнего перенесённого, поэтому `append` остатка корректен. В конце перенесены все элементы ровно по одному разу.

## Код Go 1.26.5

`solution.go`:

```go
package mergearrays

import "errors"

func MergeSorted(a, b []int) ([]int, error) {
	if !nonDecreasing(a) || !nonDecreasing(b) {
		return nil, errors.New("inputs must be sorted in non-decreasing order")
	}

	out := make([]int, 0, len(a)+len(b))
	i, j := 0, 0
	for i < len(a) && j < len(b) {
		if a[i] <= b[j] {
			out = append(out, a[i])
			i++
		} else {
			out = append(out, b[j])
			j++
		}
	}
	out = append(out, a[i:]...)
	out = append(out, b[j:]...)
	return out, nil
}

func nonDecreasing(values []int) bool {
	for i := 1; i < len(values); i++ {
		if values[i] < values[i-1] {
			return false
		}
	}
	return true
}
```

## Table-driven tests

`solution_test.go`:

```go
package mergearrays

import (
	"reflect"
	"testing"
)

func TestMergeSorted(t *testing.T) {
	tests := []struct {
		name    string
		a, b    []int
		want    []int
		wantErr bool
	}{
		{
			name: "duplicates",
			a:    []int{1, 2, 5},
			b:    []int{1, 2, 3, 4, 6},
			want: []int{1, 1, 2, 2, 3, 4, 5, 6},
		},
		{
			name: "interleaving",
			a:    []int{4, 7, 13},
			b:    []int{3, 5, 8, 9, 11},
			want: []int{3, 4, 5, 7, 8, 9, 11, 13},
		},
		{name: "empty left", a: nil, b: []int{1, 2}, want: []int{1, 2}},
		{name: "both empty", a: nil, b: nil, want: []int{}},
		{name: "invalid order", a: []int{2, 1}, b: nil, wantErr: true},
	}

	for _, tc := range tests {
		t.Run(tc.name, func(t *testing.T) {
			got, err := MergeSorted(tc.a, tc.b)
			if (err != nil) != tc.wantErr {
				t.Fatalf("error = %v, wantErr %v", err, tc.wantErr)
			}
			if !tc.wantErr && !reflect.DeepEqual(got, tc.want) {
				t.Fatalf("got %v, want %v", got, tc.want)
			}
		})
	}
}
```

## Сложность

Валидация и merge вместе занимают `O(M+N)` времени. Результат требует `O(M+N)` памяти; помимо возвращаемого слайса используется `O(1)` дополнительной памяти.

## Trade-offs

Конкатенация с последующей сортировкой короче, но стоит `O((M+N) log(M+N))` и игнорирует уже известный порядок. In-place merge возможен, если у первого буфера есть свободная ёмкость и известна его логическая длина; тогда удобно идти с конца, но это другой контракт и mutation входа.

## Типичные ошибки

- **Цикл работает, пока оба указателя валидны, и сразу возвращает результат → теряется хвост → один вход закончился раньше → дописать оба остатка после цикла.**
- **При равенстве сдвигать оба указателя → исчезает один duplicate → merge перепутан с пересечением множеств → переносить один элемент за шаг.**
- **Не проверить предусловие → на несортированном входе выдаётся правдоподобный мусор → доказательство опирается на сортировку хвостов → валидировать или документировать доверенное предусловие.**
