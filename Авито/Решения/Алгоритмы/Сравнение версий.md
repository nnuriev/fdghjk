# Сравнение версий

## TL;DR

Разбиваем версии на числовые сегменты, убираем ведущие нули и сравниваем числа без преобразования в машинный integer: сначала длину, затем лексикографически. Отсутствующие хвостовые сегменты считаются нулевыми.

Знак результата **намеренно необычный**: `1`, если `left < right`; `-1`, если `left > right`; `0`, если версии равны. Это обратный знак относительно большинства comparator API.

## Нормализованное условие и контракт

Корректная версия соответствует форме `v[0-9]+(\.[0-9]+)*`: латинская строчная `v`, затем один или больше непустых десятичных сегментов. Ведущие нули в сегменте разрешены и не влияют на значение. Версии сравниваются слева направо по числовым значениям сегментов; недостающие справа сегменты равны нулю.

Результат:

| Отношение | Результат |
| --- | ---: |
| `left < right` | `1` |
| `left == right` | `0` |
| `left > right` | `-1` |

Некорректная строка возвращает ошибку. Длина числового сегмента не ограничена разрядностью `int64`.

### Неоднозначности исходника

- «less-формат» обычно ассоциируется с boolean или отрицательным знаком для меньшего значения, но примеры однозначно задают обратное: `v11.22.44 < v11.22.45` возвращает `1`, а `v1.12.3 > v1.3.4` — `-1`.
- «Комбинация точек и чисел» могла бы разрешить пустой сегмент. Здесь `v1..2`, `v.1` и `v1.` отвергаются, потому что у пустого сегмента нет числового значения.
- `v1` и `v1.0.0` считаются равными по правилу дополнения младшими нулями из исходника.

## Ментальная модель

Версия — вектор неотрицательных целых бесконечной длины, у которого после последнего записанного компонента идут нули. Сравнение — lexicographic order этого вектора. Сегмент после удаления ведущих нулей сравнивается как большое число по длине и байтам.

## Алгоритм и корректность

1. Валидируем префикс `v`, непустые сегменты и ASCII-цифры.
2. Нормализуем каждый сегмент: `00012 → 12`, `000 → 0`.
3. Идём до максимального числа сегментов, подставляя `0` за отсутствующие.
4. Первый различающийся сегмент определяет отношение всей версии; если различий нет, возвращаем `0`.

Для нормализованных десятичных строк без ведущих нулей большее число имеет либо большую длину, либо при равной длине лексикографически большую строку. Поэтому сравнение сегментов численно корректно без риска overflow. Lexicographic order векторов определяется первым отличием; дополнение нулями реализует правило эквивалентности хвоста.

## Код Go 1.26.5

`solution.go`:

```go
package versions

import (
	"errors"
	"strings"
)

func CompareVersions(left, right string) (int, error) {
	a, err := parseVersion(left)
	if err != nil {
		return 0, err
	}
	b, err := parseVersion(right)
	if err != nil {
		return 0, err
	}

	count := len(a)
	if len(b) > count {
		count = len(b)
	}
	for i := 0; i < count; i++ {
		x, y := "0", "0"
		if i < len(a) {
			x = a[i]
		}
		if i < len(b) {
			y = b[i]
		}
		if relation := compareDecimal(x, y); relation < 0 {
			return 1, nil
		} else if relation > 0 {
			return -1, nil
		}
	}
	return 0, nil
}

func parseVersion(version string) ([]string, error) {
	if len(version) < 2 || version[0] != 'v' {
		return nil, errors.New("version must start with v and contain a segment")
	}
	segments := strings.Split(version[1:], ".")
	for i, segment := range segments {
		if segment == "" {
			return nil, errors.New("version contains an empty segment")
		}
		for j := 0; j < len(segment); j++ {
			if segment[j] < '0' || segment[j] > '9' {
				return nil, errors.New("version contains a non-digit")
			}
		}
		first := 0
		for first < len(segment)-1 && segment[first] == '0' {
			first++
		}
		segments[i] = segment[first:]
	}
	return segments, nil
}

func compareDecimal(left, right string) int {
	if len(left) < len(right) {
		return -1
	}
	if len(left) > len(right) {
		return 1
	}
	if left < right {
		return -1
	}
	if left > right {
		return 1
	}
	return 0
}
```

## Table-driven tests

`solution_test.go`:

```go
package versions

import "testing"

func TestCompareVersions(t *testing.T) {
	tests := []struct {
		name        string
		left, right string
		want        int
		wantErr     bool
	}{
		{name: "left is less returns positive", left: "v11.22.44", right: "v11.22.45", want: 1},
		{name: "equal", left: "v11.22.44", right: "v11.22.44", want: 0},
		{name: "trailing zero segments", left: "v11.22.44", right: "v11.22.44.0", want: 0},
		{name: "numeric not lexical", left: "v1.12.3", right: "v1.3.4", want: -1},
		{name: "leading zeroes", left: "v01.002", right: "v1.2.0", want: 0},
		{
			name: "segment larger than int64",
			left: "v999999999999999999999999", right: "v1000000000000000000000000", want: 1,
		},
		{name: "empty segment", left: "v1..2", right: "v1.2", wantErr: true},
		{name: "wrong prefix", left: "1.2", right: "v1.2", wantErr: true},
	}

	for _, tc := range tests {
		t.Run(tc.name, func(t *testing.T) {
			got, err := CompareVersions(tc.left, tc.right)
			if (err != nil) != tc.wantErr {
				t.Fatalf("error = %v, wantErr %v", err, tc.wantErr)
			}
			if !tc.wantErr && got != tc.want {
				t.Fatalf("got %d, want %d", got, tc.want)
			}
		})
	}
}
```

Байтовая проверка здесь намеренна: grammar разрешает только ASCII `0..9`; общая модель bytes/runes раскрыта в [[60 Go/Строки, байты, руны и UTF-8|заметке о строках]].

## Сложность

Парсинг и сравнение занимают `O(L+R)` времени, где `L`, `R` — длины строк. `strings.Split` и слайсы сегментов требуют `O(L+R)` памяти. Преобразования в bounded integers нет.

## Trade-offs

`strconv.ParseUint` делает код сравнения короче, но вводит искусственный предел разрядности и ошибки overflow для формально допустимой версии. Потоковый parser с двумя индексами снизил бы дополнительную память до `O(1)`, однако заметно усложнил бы валидацию. Нормализованные строки дают хороший баланс для интервью.

## Типичные ошибки

- **Сравнивать строки целиком → `12 < 3` лексикографически → перепутаны текст и число → сравнивать сегменты численно.**
- **Вернуть стандартный знак comparator → пример получает `-1` вместо `1` → контракт имеет обратную полярность → зафиксировать таблицу до кода.**
- **Остановиться на длине короткой версии → `v1` ошибочно отличается от `v1.0` → хвостовые нули не учтены → идти до максимальной длины.**
- **Парсить сегмент в `int64` → длинная версия отвергается из-за реализации, а не grammar → сравнивать нормализованные decimal strings.**
- **Передать результат напрямую в `slices.SortFunc` → сортировка инвертируется → `SortFunc` ожидает отрицательное значение для `a<b` → адаптировать знак отдельно.**
