Программирование

Цель интервью

  • Проверить навыки кандидата в написании и отлаживании (при необходимости) кода.
  • Проверить умение кандидата обеспечить работу решения не только для предоставленных примеров (корнер-кейсы)
  • Проверить знания кандидата базовых структур данных (список, стэк, хэш-таблица и тд) и алгоритмов (например, рекурсивный, линейный и тд)
  • Проверить знание и понимание кандидатом алгоритмической сложности

В результате этого интервью определяем уровень кандидата: ниже начального, начальный, средний или продвинутый.

Также обращаем внимание на то, как кандидат общается (отмечаем в отчете агрессию в сторону формата вопросов/интервью, мат и тд)

Как определить уровень кандидата

Ниже начального

  • Не решил ни одной задачи
  • Не смог самостоятельно придумать алгоритм для решения

Начальный

  • Решил easy-задачу
  • Придумал алгоритм по easy задаче и перенес его в код. Подсказки возможны (набор данных, на которых сломается решение)
  • Может посчитать алгоритмическую сложность своего решения (хотя бы на уровне “цикл в цикле – это O(N^2)”)
  • Знает алгоритмическую сложность операций с распространенными структурами данных (динамический массив, хеш-таблица)
  • Может отладить написанное в уме, в ходе дебага находит ошибки
  • Способен придумать корнер кейсы

Средний

  • Решил easy и medium задачу
  • Придумал алгоритм по easy задаче и перенес его в код. Приемлемы подсказки, не влияющие на итоговый алгоритм (например, наводящие вопросы про работу со структурами: как найти элемент в списке? как быстро это будет работать?)
  • Придумал алгоритм по medium задаче и перенес его в код. Приемлемы подсказки, не влияющие на итоговый алгоритм
  • Оперирует алгоритмической сложностью Правильно назвал сложность алгоритма Правильно определил потребление памяти
  • Знает алгоритмическую сложность операций с распространенными структурами данных (динамический массив, хеш-таблица)
  • Способен придумать корнер кейсы

Продвинутый

  • То же самое, что и для среднего
  • Решил все задачи оптимально с точки зрения временной/пространственной сложности
  • Задачи не требуют сильного рефакторинга
  • Решил все задачи без подсказок
  • Пояснял свои решения, рассуждал в процессе
  • Свободно оперирует алгоритмической сложностью
    • Знает сложности основных сортировок (quick, merge, bubble)
    • Различает сложность в худшем/лучшем/среднем случае
    • Понимает про константы и доминанты

До интервью

  • Создаем новый отчет;
  • Заполняем поля «Кандидат», «Интервьюер(ы)» и «Дата».

Как начать интервью

«Привет, меня зовут %ИМЯ%, я работаю в Авито в %отделе подсчета денег%. Ничего если я буду на ты?

Я проведу у тебя секцию Программирования. Она рассчитана на час времени, в течение которого мы решим одну или несколько задач. Тебе нужно написать правильное решение, в конце задачи оценить сложность по времени и по памяти.

Будет хорошо, если во время решения ты будешь размышлять вслух и комментировать свои шаги. От тебя не требуется точно знать названия функций или синтаксис языка. Но если ты используешь что-то из стандартной библиотеки, типа сортировки, должен объяснить допущения по сложности.

Сейчас я скину ссылку на блокнотик avito.code, где мы будем писать код.»

Во время интервью

Предлагаем решить несколько задач из библиотеки задач, учитывая уровень кандидата и требования к вакансии (одна задача уровня easy и одна задача medium). Если кандидат использует стандартную библиотеку, готовые структуры данных и т.п, то он должен уметь объяснить, как оно работает внутри "на пальцах".

Оценка решения кандидата

Рекомендации:

  • Не требуем изначально максимально эффективного решения. Сначала надо просто решить задачу. Далее можно оценить текущее решение и подвести кандидата к поиску более оптимального.
  • Обращаем внимание на:
    • Ясность мышления / умение объяснить, как работает написанный код
    • Наименование переменных / функций (понятное, не слишком длинное, не слишком короткое)
    • Форматирование кода
  • НЕ обращаем внимание на:
    • Точки с запятыми
    • Единичные опечатки
    • Модные фичи языка
    • Ошибки при рефакторинге (где-то что-то забыл переименовать)
    • Если забыл(а) имя функции из стандартной библиотеки - подсказываем или выдумываем

Артефактом решения задач должен стать полный текст задач в файле отчета и решения, предложенные кандидатом, а также ответы кандидата на дополнительные вопросы.

Как закончить интервью

«У меня вопросов больше нет. Может, ты хочешь что-нибудь спросить у меня? Если нет, то я могу рассказать, как у нас всё устроено в Авито или что будет дальше на следующих этапах.»

Отвечаем на вопросы, объясняем остальные секции и примерные сроки; попутно рассказываем, за что вы сами любите Авито – это помогает продать вакансию

«Спасибо за то, что пришел(ла), было приятно с тобой попрограммировать, удачи на следующих этапах.»

После интервью

  • «Причесываем» заметки по ответам кандидата;
  • Заполняем поля «Ссылка на код», «Оценка уровня» и оставляем комментарий на основе чего было принято это решение;
  • Нажимаем «Сохраняить отчет»;
  • Переходим на детальную страницу интервью по ссылке в календаре (приглашении), добавляем ссылку на отчет в поле «Ссылка на итоги», выставляем «Уровень» и завершаем интервью.

Рекомендуемые задачи

Условие задачи

Мы в Авито любим проводить соревнования, — недавно мы устроили чемпионат по шагам. И вот настало время подводить итоги!

Необходимо определить userIds участников, которые прошли наибольшее количество шагов steps за все дни, не пропустив ни одного дня соревнований.

Пример

# Пример 1
# ввод
statistics = [
        [{ userId: 1, steps: 1000 }, { userId: 2, steps: 1500 }],
        [{ userId: 2, steps: 1000 }]
]

# вывод
champions = { userIds: [2], steps: 2500 }

# Пример 2
statistics = [
        [{ userId: 1, steps: 2000 }, { userId: 2, steps: 1500 }],
        [{ userId: 2, steps: 4000 }, { userId: 1, steps: 3500 }]
]

# вывод
champions = { userIds: [1, 2], steps: 5500 }

Алгоритмическая сложность

Т.к. в условии не оговорено ограничение на массив statistics, правильными являются оба ответа (главное, чтобы кандидат объяснил, что он подразумевает под N и K в каждом случае). Ожидаемая сложность: * O(NK), где N - количество дней(строк в массиве statistics), K - максимальное количество участников в день.
или * O(N), где N - суммарное количество записей в массиве за все дни.

Способы решения

Проход по всем элементам с записью в мапу, затем фильтрация мапы по условиям задачи.

Возможные вопросы

Спрашивать об оптимизациях, если в мапу кладутся абсолютно все участники, не принимавшие участия в предыдущих днях.

Например, если в первый день участвовал 1 человек, а во второй 10 000, то нет смысла класть их всех в мапу.

Основные ошибки/корнер-кейсы

Может быть несколько победителей с одинаковым количеством шагов. Победителей может не быть вообще(если каждый участник участвововал не во все дни). На каждом шаге проверять, что участник был в предыдущем через includes(неоптимально). Использование сортировки(неоптимально).

Решение + тесты

package main

import (
    "fmt"
    "reflect"
    "sort"
)

type Statistic struct {
    UserID int
    Steps  int
}

type Result struct {
    UserIDs []int
    Steps   int
}

func getChampions(statistics [][]Statistic) Result {
    if len(statistics) == 0 {
        return Result{}
    }

    type UserStatistic struct {
        DaysIn   int
        StepsSum int
    }

    possibleChampions := make(map[int]UserStatistic, len(statistics[0]))
    for _, userStat := range statistics[0] {
        possibleChampions[userStat.UserID] = UserStatistic{
            DaysIn:   1,
            StepsSum: userStat.Steps,
        }
    }

    for day := 1; day < len(statistics); day++ {
        for _, userDayStatistic := range statistics[day] {
            userID := userDayStatistic.UserID
            if dayStatistic, found := possibleChampions[userID]; found {
                dayStatistic.DaysIn++
                dayStatistic.StepsSum += userDayStatistic.Steps
                possibleChampions[userID] = dayStatistic

            }
        }
    }

    maxSteps := -1
    for _, stat := range possibleChampions {
        if stat.DaysIn == len(statistics) {
            maxSteps = max(maxSteps, stat.StepsSum)
        }
    }

    winners := make([]int, 0, len(possibleChampions))
    for userID, stat := range possibleChampions {
        if stat.StepsSum == maxSteps && stat.DaysIn == len(statistics) {
            winners = append(winners, userID)
        }
    }
    if len(winners) == 0 {
        return Result{}
    }

    return Result{
        UserIDs: winners,
        Steps:   maxSteps,
    }
}

func main() {
    fmt.Println("empty", reflect.DeepEqual(getChampions(nil), Result{}))
    fmt.Println("case 1", reflect.DeepEqual(
        getChampions(
            [][]Statistic{

                {{UserID: 2, Steps: 1000}},
            },
        ),
        Result{
            UserIDs: []int{2},
            Steps:   2500,
        },
    ))
    fmt.Println("case 2", reflect.DeepEqual(
        getChampions(
            [][]Statistic{
                {{UserID: 1, Steps: 2000}, {UserID: 2, Steps: 1500}},
                {{UserID: 2, Steps: 1000}, {UserID: 3, Steps: 4000}},
            },
        ),
        Result{
            UserIDs: []int{2},
            Steps:   2500,
        },
    ))
    fmt.Println("case 3", reflect.DeepEqual(
        getChampions(
            [][]Statistic{
                {{UserID: 1, Steps: 2000}, {UserID: 2, Steps: 1500}},
                {{UserID: 2, Steps: 1000}, {UserID: 1, Steps: 3000}},
            },
        ),
        Result{
            UserIDs: []int{1},
            Steps:   5000,
        },
    ))
    // А тут у нас может быть флакующий тест, и так как мы ничего не импортируем нормального, то напишем так:
    case4Answer := getChampions(
        [][]Statistic{

JavaScript

Решение

/**
 * 
 * @param {Array<Array<{userId: number; steps: number;}>>} statistics 
 * @returns {{userIds: number[]; steps: number;} | null}
 */
function getChampions(statistics = []) {
    // Только те, кто был с самого начала, могут победить
    const possibleChampions = statistics[0].reduce((acc, next) => {
        acc[next.userId] = {
            daysIn: 1,
            steps: next.steps
        };
        return acc;
    }, {});

    for (let i = 1; i < statistics.length; i++) {
        for (const result of statistics[i]) {
            if (possibleChampions.hasOwnProperty(result.userId) && possibleChampions[result.userId].daysIn === i) {
                possibleChampions[result.userId].steps += result.steps;
                possibleChampions[result.userId].daysIn += 1;
            }
        }
    }

    // Берем только тех, кто каждый день участвовал
    const activeChampions = Object.entries(possibleChampions).filter(([,{ daysIn }]) => daysIn === statistics.length);
    const maxSteps = Math.max(...activeChampions.map(([,{ steps }]) => steps));

    if (maxSteps === Number.NEGATIVE_INFINITY) {
        return null;
    }

    const winners = {
        userIds: [],
        steps: maxSteps
    };

    for (let champion of activeChampions) {
        const [userId, {steps}] = champion
        if (steps === maxSteps) {
            winners.userIds.push(+userId);
        }
    }
}

Kotlin

Решение

data class UserStatistics(var steps: Int = 0, var amountDay: Int = 0)
class DayStatistics(val userId: Int, val steps: Int)
class Winners(val userIds: IntArray = intArrayOf(), val steps: Int = 0)

fun getChampions(statistics: Array<Array<DayStatistics>>): Winners {
    if (statistics.isEmpty()) {
        return Winners()
    }
    val map = HashMap<Int, UserStatistics>()
    for (i in statistics.indices) { //аналог i >= 0 && i < statistics.size
        for (j in statistics[i].indices) {

            val dayStatistics = statistics[i][j]
            if (map[dayStatistics.userId] == null && i == 0) {
                map[dayStatistics.userId] = UserStatistics()
            }
            map[dayStatistics.userId]?.let {
                if (it.amountDay != i) {
                    map.remove(dayStatistics.userId)
                } else {
                    it.steps += dayStatistics.steps
                    it.amountDay++
                }
            }
        }
    }
    var maxSteps = 0
    var resUserIds = ArrayList<Int>()
    for (user in map.entries) {
        if (user.value.amountDay == statistics.size) {
            if (maxSteps == user.value.steps) {
                resUserIds.add(user.key)

            } else if (maxSteps < user.value.steps) {
                resUserIds = arrayListOf(user.key)
                maxSteps = user.value.steps
            }
        }
    }

    return Winners(resUserIds.toIntArray(), maxSteps)
}

    let steps: Int

struct Champions {

func getChampions(_ stats: [DailyStats]) -> Champions? {
    struct Competitor {
        var totalSteps: Int
        var daysIn: Int
    }

    var daysPassed = 1
    var competitors = [Int: Competitor]()

    stats.first?.forEach { userStat in
        competitors[userStat.userId] = Competitor(
            totalSteps: userStat.steps,
            daysIn: daysPassed
        )
    }

    guard !competitors.isEmpty
    else { return nil }

    for dayStat in stats.dropFirst() {
        daysPassed += 1
        for userStat in dayStat {
            if let competitor = competitors[userStat.userId] {
                competitors[userStat.userId] = Competitor(
                    totalSteps: competitor.totalSteps + userStat.steps,
                    daysIn: daysPassed
                )
            }
        }

        // Filtering out competitors who missed the day
        competitors = competitors.filter { competitor in
            competitor.value.daysIn == daysPassed
        }
    }

    guard !competitors.isEmpty
    else { return nil }

    var maxSteps = 0
    var winners = [Int]()

    for competitor in competitors {
        if competitor.value.totalSteps > maxSteps {
            winners = [competitor.key]
            maxSteps = competitor.value.totalSteps

        } else if competitor.value.totalSteps == maxSteps {
            winners.append(competitor.key)
        }
    }

    return Champions(userIds: winners.sorted(), totalSteps: maxSteps)
}

        var totalSteps: Int
        var daysIn: Int
    }

    let firstDayCompetitors = stats.first?.reduce(into: [Int: Competitor]()) { competitors, userStat in
        competitors[userStat.userId] = Competitor(
            totalSteps: userStat.steps,
            daysIn: 1
        )
    }

    guard let firstDayCompetitors, !firstDayCompetitors.isEmpty
    else { return nil }

    let competitors = stats.dropFirst().reduce(into: firstDayCompetitors) { competitors, dayStat in
        dayStat.forEach { userStat in
            if let competitor = competitors[userStat.userId] {
                competitors[userStat.userId] = Competitor(
                    totalSteps: competitor.totalSteps + userStat.steps,
                    daysIn: competitor.daysIn + 1
                )
            }
        }
    }

    // Filtering out competitors who missed some days
    let totalDays = stats.count
    let allDaysCompetitors = competitors.filter { competitor in
        competitor.value.daysIn == totalDays
    }

    guard !allDaysCompetitors.isEmpty
    else { return nil }

    let maxSteps = allDaysCompetitors.reduce(-1) { maxSteps, competitor in
        max(maxSteps, competitor.value.totalSteps)
    }

    let winners = allDaysCompetitors.compactMap { userId, competitor in
        competitor.totalSteps == maxSteps ? userId : nil
    }

    return Champions(userIds: winners.sorted(), totalSteps: maxSteps)
}

Условие задачи

На Авито размещено множество товаров, каждый из которых представлен числом. У каждого покупателя есть потребность в товаре, также выраженная числом. Если точного товара нет, покупатель выбирает ближайший по значению товар, что вызывает неудовлетворённость (dissatisfaction), равную разнице между его потребностью и купленным товаром. Количество каждого товара не ограничено, и один товар могут купить несколько покупателей. Рассчитайте суммарную неудовлетворённость всех покупателей.

Нужно написать функцию, которая примет на вход два массива: массив товаров и массив потребностей покупателей, вычислит сумму неудовлетворённостей всех покупателей и вернет результат в виде числа.

Пример

# Пример
# ввод
goods = [8, 3, 5]
buyerNeeds = [5, 6]
# вывод
res = 1 # первый покупатель покупает товар 5 и его неудовлетворённость = 0, второй также покупает товар 5 и его неудовлетворённость = 6-5 = 1

Алгоритмическая сложность

Ожидаемая сложность O(N*log(N)). Ожидаемое потребление памяти O(1), дополнительной памяти, кроме входных массивов не выделяется.

Бейзлайн по времени O(N^2)

Способы решения

  • Полный перебор. Для каждой потребности проходим по массиву товаров и выбираем наиболее близкий по значению. Сложность O(n*m), где n - кол-во товаров, m - кол-во покупателей
  • Два указателя. Отсортировать оба массива. Пройти синхронно по обоим массивам, для каждой потребности определяя наилучший товар. Для каждой потребности двиагем указатель по товарам вперёд, пока следующий товар для неё лучше или равен текущему. Сложность O(max(n, m)log(max(n, m))). Более точно nlog(n) + m*log(m). Где n - кол-во товаров, m - кол-во покупателей
  • Бинпоиск. Отсортировать массив товаров. Пройти массиву потребностей, для каждой потребности находить в товар бинпоиском. Сложность O(max(n, m)log(n)). Более точно nlog(n) + m*log(n). Где n - кол-во товаров, m - кол-во покупателей

Основные ошибки

Заполняется по мере работы кандидатов с задачей

func dissatisfaction(_ goods: [Int], _ buyerNeeds: [Int]) -> Int {
    func getClosest(to target: Int, in array: [Int]) -> Int {
        var (left, right) = (0, array.count-1)

        while left + 1 < right {
            let mid = (left + right) / 2
            if array[mid] < target {
                left = mid
            } else {
                right = mid
            }
        }

        return abs(array[left] - target) < abs(array[right] - target)
            ? array[left]
            : array[right]
    }

    let sortedGoods = goods.sorted()

    return buyerNeeds.reduce(0) { dissatisfaction, buyerNeed in
        dissatisfaction + abs(getClosest(to: buyerNeed, in: sortedGoods) - buyerNeed)
    }
}

def binsearch_closest(arr, target):
    """возвращает ближайшее к target из массива arr"""
    left, right = 0, len(arr) - 1

    while left + 1 < right:
        mid = (left + right) // 2
        if arr[mid] < target:
            left = mid
        else:
            right = mid

    if abs(arr[left] - target) < abs(arr[right] - target):
        return arr[left]
    return arr[right]

import bisect

def f(goods, buyer_needs):
    goods.sort()
    total_dissatisfaction = 0

    for need in buyer_needs:
        pos = bisect.bisect_left(goods, need)

        if pos == 0:
            closest = goods[0]
        elif pos == len(goods):
            closest = goods[-1]
        else:
            before = goods[pos - 1]
            after = goods[pos]
            if after - need < need - before:
                closest = after
            else:
                closest = before

        total_dissatisfaction += abs(need - closest)

    return total_dissatisfaction
