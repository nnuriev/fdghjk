# Геопространственные индексы и поиск ближайших объектов

## Вопрос

Объясните тему «Геопространственные индексы и поиск ближайших объектов»: какие гарантии даёт механизм и какой ценой для чтения, записи и эксплуатации?

## Короткий ориентир

Геопространственный индекс редко отвечает на бизнес-вопрос целиком. Он быстро строит небольшой набор candidates по bounding boxes или grid cells, после чего система вычисляет точное пространственное отношение или расстояние. Эта двухфазность — главный инвариант: approximate filter может возвращать лишних кандидатов, но не должен терять подходящие объекты.

В PostgreSQL/PostGIS R-tree-подобный поиск реализован через GiST operator classes: индекс хранит bounding boxes, а `ST_DWithin` и KNN-оператор `<->` используют его для сокращения чтения. H3 4.5.0 кодирует поверхность иерархическими шестиугольными cells; соседние cells удобны для распределённого candidate lookup, но grid distance не равен точному геодезическому расстоянию. Для движущихся объектов geometry и index недостаточно: нужны versioned location updates, удаление «призраков» из прежних cells, TTL и защита от hot cells.
