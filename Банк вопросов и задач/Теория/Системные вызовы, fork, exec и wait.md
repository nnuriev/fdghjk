# Системные вызовы, fork, exec и wait

## Вопрос

Как user process переходит в kernel, что наследуется при `fork`, что заменяет `exec` и зачем родителю `wait`/`waitpid`?

## Короткий ориентир

System call — контролируемый переход из user mode в kernel для операции над ресурсом ОС. `fork` создаёт новый process с отдельным address space, логически унаследованным состоянием и file descriptors; современные системы обычно откладывают физическое копирование страниц через copy-on-write. `exec` не создаёт ещё один process, а заменяет image текущего. `wait`/`waitpid` позволяют родителю получить статус завершения и освободить kernel bookkeeping дочернего процесса.

Ownership descriptors дополнительно раскрыт в [[Банк вопросов и задач/Теория/Файловые дескрипторы|карточке файловых дескрипторов]].
