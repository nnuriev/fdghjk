# Индексы и цена чтения и записи

## Вопрос

Объясните тему «Индексы и цена чтения и записи»: какие гарантии даёт механизм и какой ценой для чтения, записи и эксплуатации?

## Короткий ориентир

Индекс хранит дополнительное представление ключей, по которому PostgreSQL может найти небольшой диапазон строк, выдать совместимый порядок или проверить uniqueness без полного сканирования heap. Он ускоряет только те чтения, для которых predicate, operators, ordering и selectivity подходят access method.

Каждый индекс оплачивается на записи: новые entries, page splits, WAL, cache pressure, vacuum и иногда потеря HOT update. Поэтому вопрос звучит не «какой индекс добавить», а «какой измеренный read path окупает постоянную write amplification».
