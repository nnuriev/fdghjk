# Доказательство корректности алгоритма

## Вопрос

Объясните тему «Доказательство корректности алгоритма»: как устроен механизм, какие инварианты определяют поведение и где проходят практические границы?

## Короткий ориентир

Корректность — это утверждение вида: при выполнении предусловий алгоритм завершается и возвращает результат, удовлетворяющий postcondition. Тест показывает поведение на выбранных входах; proof объясняет, почему ошибка невозможна во всём заявленном классе.

На coding interview обычно хватает короткого proof sketch. Для цикла назовите инвариант, покажите initialization, maintenance и termination. Для recursion используйте индукцию по размеру подзадачи; для greedy — exchange или stays-ahead argument; для graph algorithms — свойство frontier, cut или relaxation. Доказательство должно опираться ровно на тот механизм, который реализован в коде.

- В каких условиях подход полезен, а в каких опасен?
