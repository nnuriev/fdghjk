# Процессор, кеши, когерентность и NUMA

## Вопрос

Как pipeline, cache hierarchy, coherence, memory ordering и NUMA placement влияют на наблюдаемую производительность и корректность concurrent-кода?

## Короткий ориентир

Процессор скрывает latency через pipeline, out-of-order и speculative execution, а cache hierarchy сокращает число обращений к RAM. Несколько cores требуют coherence protocol для согласования cache lines, но coherence не заменяет memory model и синхронизацию программы. На NUMA-машине цена доступа дополнительно зависит от того, на каком node размещены CPU и страницы памяти; pinning только потока без placement данных не гарантирует locality.
