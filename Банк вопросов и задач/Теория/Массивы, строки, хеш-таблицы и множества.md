# Массивы, строки, хеш-таблицы и множества

## Вопрос

Объясните тему «Массивы, строки, хеш-таблицы и множества»: как устроен механизм, какие инварианты определяют поведение и где проходят практические границы?

## Короткий ориентир

Вопрос заметки: как выбрать представление для последовательности, текста и membership-запросов так, чтобы нужная операция была дешёвой, а скрытые предпосылки оставались явными?

Массив хранит элементы в индексируемой последовательности и даёт `Θ(1)` random access. За вставку в середину приходится сдвигать хвост, то есть платить `Θ(n)`. Строка добавляет кодировку и семантику символов: индекс по byte не обязан совпадать с пользовательским символом. Hash map реализует соответствие `key → value`, set оставляет только принадлежность; при контролируемом load factor операции обычно имеют expected `Θ(1)`, но worst-case остаётся `Θ(n)`.

Практическое правило: сначала выпишите операции и допустимый key universe. Плотные небольшие integer keys часто лучше хранить в массиве, порядок и range queries требуют упорядоченной структуры, а hash table выигрывает на точечных lookup без требования порядка.
