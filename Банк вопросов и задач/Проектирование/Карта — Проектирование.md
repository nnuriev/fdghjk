# Проектирование

## Назначение

Задачи High-Level Design и Low-Level Design с требованиями, оценкой нагрузки, API, моделью данных, failure modes и архитектурными trade-offs.

## Входные знания

Используйте [[50 Проектирование систем/Методика System Design интервью|методику System Design интервью]] и [[50 Проектирование систем/Карта — Проектирование систем|карту проектирования систем]].

## Маршрут

Для каждой задачи сначала фиксируйте требования и ограничения, затем нагрузку, состояние и критические потоки. Конкретные технологии выбираются после этих шагов.
