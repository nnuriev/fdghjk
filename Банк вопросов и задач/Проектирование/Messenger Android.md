# Messenger (Android)

## TL;DR

В материалах Avito сохранилось только название «Messenger (Android)». Функциональные требования, нагрузка, API и ожидаемая глубина проектирования отсутствуют, поэтому архитектурное решение не восстанавливается.

## Контекст и ментальная модель

Наблюдаемые варианты: **«Messenger (Android)»**. Они перечислены в [[Авито/roadmap#Вне выбранного scope|разделе «Вне выбранного scope»]].

Источник относит название к клиентской Android-архитектуре вне выбранного Go-backend scope. Категория «Проектирование» здесь только хранит технический пробел источника.

## Требования

### Функциональные

Неизвестны. Нужно восстановить:

- мобильный продуктовый сценарий;
- границы Android-клиента и backend;
- offline model, sync, storage, networking и критерии оценки.

### Нефункциональные и SLO

Неизвестны: источник не сохраняет нагрузку, latency, availability, durability, consistency и resource limits.

### Вне scope

Неизвестно до восстановления исходного условия. Сам title-only пункт уже помечен источником как находящийся вне выбранного Go-backend scope.
