# Библиотека кэширования файлов

## TL;DR

В материалах Avito сохранилось только название «Библиотека кэширования файлов». Функциональные требования, нагрузка, API и ожидаемая глубина проектирования отсутствуют, поэтому архитектурное решение не восстанавливается.

## Контекст и ментальная модель

Наблюдаемые варианты: **«Библиотека кэширования файлов»**. Они перечислены в [[Авито/roadmap#System design и проектирование|разделе «System design и проектирование»]].

После восстановления API пригодятся [[10 Основы CS/Файловая система и буферизация|модель файловой системы и буферизации]] и [[50 Проектирование систем/Проектирование и реализация in-memory cache в Go|разбор in-memory cache]].

## Требования

### Функциональные

Неизвестны. Нужно восстановить:

- API библиотеки и типы файлов;
- политика eviction, TTL и лимиты диска или памяти;
- concurrency, consistency и обработка повреждённых данных.

### Нефункциональные и SLO

Неизвестны: источник не сохраняет нагрузку, latency, availability, durability, consistency и resource limits.

### Вне scope

Неизвестно до восстановления исходного условия.
