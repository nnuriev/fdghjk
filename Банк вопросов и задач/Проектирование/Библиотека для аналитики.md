# Библиотека для аналитики

## TL;DR

В материалах Avito сохранилось только название «Библиотека для аналитики». Функциональные требования, нагрузка, API и ожидаемая глубина проектирования отсутствуют, поэтому архитектурное решение не восстанавливается.

Одинаковое название встречается в исходном списке дважды; повтор объединён в одну карточку.

## Контекст и ментальная модель

Наблюдаемые варианты: **«Библиотека для аналитики»**. Они перечислены в [[Авито/roadmap#System design и проектирование|разделе «System design и проектирование»]].

После восстановления условия его можно сравнить с [[50 Проектирование систем/Проектирование event ingestion pipeline|проектированием event ingestion pipeline]].

## Требования

### Функциональные

Неизвестны. Нужно восстановить:

- API и типы событий;
- гарантии доставки, buffering и backpressure;
- схема, batching, отказоустойчивость и требования к ресурсам.

### Нефункциональные и SLO

Неизвестны: источник не сохраняет нагрузку, latency, availability, durability, consistency и resource limits.

### Вне scope

Неизвестно до восстановления исходного условия.
