## Ментальная модель

Матрица — это не отдельный алгоритм, а способ адресовать двумерное состояние. Почти все задачи курса сводятся к одному из четырёх приёмов:

1. вывести формулу множества клеток: `i == j`, `i + j == n - 1`;
2. переставить координаты: `(row, col) → (col, row)`;
3. кодировать constraint group: строка, столбец, box;
4. поддерживать ещё не обработанный прямоугольник: `top`, `bottom`, `left`, `right`.

![[90 Вложения/CurseHunter/7254/Кадры/001-matrix-indexes.jpg|720]]

Для прямоугольной матрицы `n × m` допустимы `0 ≤ row < n`, `0 ≤ col < m`. Формула побочной диагонали `row + col == n - 1` относится к квадрату `n × n`; у прямоугольника сначала надо определить, что считать диагональю.

## 1. Matrix Diagonal Sum — LeetCode 1572

### Условие

Дана квадратная матрица. Вернуть сумму главной и побочной диагоналей, не учитывая центральную клетку дважды.

### Ожидаемое рассуждение

Для каждой строки `i` взять `mat[i][i]` и `mat[i][n-1-i]`. При нечётном `n` обе координаты совпадают в `i == n/2`, поэтому после цикла один раз вычесть центр.

```go
func diagonalSum(mat [][]int) int {
	sum := 0
	n := len(mat)
	for i := 0; i < n; i++ {
		sum += mat[i][i]
		sum += mat[i][n-1-i]
	}
	if n%2 == 1 {
		sum -= mat[n/2][n/2]
	}
	return sum
}
```

![[90 Вложения/CurseHunter/7254/Кадры/002-diagonal-sum.jpg|720]]

### Инвариант и сложность

После итерации `i` в `sum` учтены обе диагональные клетки строк `0..i`; центр пока может быть учтён дважды. Time `O(n)`, auxiliary space `O(1)`.

### Follow-up

- Убрать финальную коррекцию: добавлять побочную клетку, только если `i != n-1-i`.
- Обобщить на ragged input: сначала определить контракт; официальный input всегда квадратный.
- Найти overflow: ограничения LeetCode помещаются в `int`, в общем API может понадобиться `int64`.

## 2. Transpose Matrix — LeetCode 867

### Условие

Для матрицы `rows × cols` построить `cols × rows`, где `result[col][row] = matrix[row][col]`.

```go
func transpose(matrix [][]int) [][]int {
	rows := len(matrix)
	cols := len(matrix[0])
	result := make([][]int, cols)
	for col := 0; col < cols; col++ {
		result[col] = make([]int, rows)
		for row := 0; row < rows; row++ {
			result[col][row] = matrix[row][col]
		}
	}
	return result
}
```

![[90 Вложения/CurseHunter/7254/Кадры/003-transpose.jpg|720]]

### Сложность и важная поправка

Time `O(rows·cols)`. Возвращаемая матрица занимает `O(rows·cols)`. Auxiliary space можно назвать `O(1)` только по соглашению «не считать output»; слайд курса с безусловным `Space = O(1)` неполон.

### Failure modes

- `matrix[0]` паникует на пустом input; официальный constraint исключает пустую матрицу.
- Ragged rows могут дать out-of-range; официальный input прямоугольный.
- In-place transpose возможен просто только для квадратной матрицы. Для прямоугольной меняется layout и длины строк.

## 3. Valid Sudoku — LeetCode 36

### Условие

Проверить заполненные клетки доски `9 × 9`: цифра `1..9` не должна повторяться в строке, столбце и box `3 × 3`. Решать Sudoku не требуется.

### Однопроходное решение

Поддерживать три таблицы `bool`: `rows[9][9]`, `cols[9][9]`, `boxes[9][9]`. Для клетки `(row, col)`:

```text
digit = board[row][col] - '1'
box = (row / 3) * 3 + col / 3
```

Если любой флаг уже поднят — constraint нарушен; иначе поднять все три.

![[90 Вложения/CurseHunter/7254/Кадры/004-valid-sudoku.jpg|720]]

### Инвариант и сложность

До обработки клетки `(row, col)` таблицы точно описывают цифры во всех уже просмотренных клетках. Time `O(9²)`, то есть константа для фиксированной доски; auxiliary space тоже константа. Для обобщённой доски `N × N` — time `O(N²)`, space `O(N²)` при таких таблицах.

### Что уточнить на интервью

- Официальное условие разрешает только `'.'` и `'1'..'9'`. Код курса пропускает всё вне диапазона цифр как пустую клетку; для внешнего API неизвестный символ надо отклонять.
- Можно заменить `[9][9]bool` на девять bitmask для строк, столбцов и boxes.
- Проверка валидности частичного состояния не доказывает, что Sudoku имеет решение или единственное решение.

## 4. Rotate Image — LeetCode 48

### Условие

Повернуть квадратную матрицу `n × n` на 90° по часовой стрелке in-place.

### Декомпозиция

Поворот равен двум простым преобразованиям:

1. transpose относительно главной диагонали;
2. reverse каждой строки.

```go
func rotate(matrix [][]int) {
	n := len(matrix)
	for row := 0; row < n; row++ {
		for col := row + 1; col < n; col++ {
			matrix[row][col], matrix[col][row] = matrix[col][row], matrix[row][col]
		}
	}
	for row := 0; row < n; row++ {
		for left, right := 0, n-1; left < right; left, right = left+1, right-1 {
			matrix[row][left], matrix[row][right] = matrix[row][right], matrix[row][left]
		}
	}
}
```

![[90 Вложения/CurseHunter/7254/Кадры/005-rotate-image.jpg|720]]

Курс начинает внутренний цикл transpose с `col := row`, поэтому каждая диагональная клетка меняется сама с собой. Это корректно, но `row + 1` точнее выражает инвариант: обрабатывается только верхний треугольник.

### Доказательство преобразования

Transpose переводит старую `(r, c)` в `(c, r)`. Reverse строки меняет column `r` на `n-1-r`, итоговая координата `(c, n-1-r)` — именно поворот по часовой стрелке.

Time `O(n²)`, auxiliary space `O(1)`. Алгоритм требует квадратной матрицы; для прямоугольной in-place контракт другой.

### Follow-up

- 90° против часовой: transpose, затем reverse порядка строк.
- 180°: reverse всех строк и затем reverse порядка строк либо менять противоположные пары.
- Решить обменом четырёх клеток по слоям и сравнить доказательство/риск off-by-one.

## 5. Spiral Matrix — LeetCode 54

### Условие

Вернуть элементы прямоугольной матрицы по часовой спирали.

### Границы

```text
top = 0, bottom = rows - 1
left = 0, right = cols - 1
```

Пока `top <= bottom && left <= right`, пройти верх слева направо, правую сторону сверху вниз, низ справа налево и левую сторону снизу вверх. После каждой стороны сдвинуть соответствующую границу.

![[90 Вложения/CurseHunter/7254/Кадры/006-spiral-matrix.jpg|720]]

### Почему нужны промежуточные проверки?

После снятия верхней строки может не остаться нижней; после снятия правого столбца может не остаться левого. Без `top <= bottom` и `left <= right` центральная строка/колонка добавится дважды. Вариант курса возвращает результат сразу после схлопывания одной из границ — это корректный способ закрыть случай `1 × k` и `k × 1`.

### Инвариант и сложность

Перед каждой итерацией все элементы вне `[top..bottom] × [left..right]` уже добавлены ровно один раз, а внутри не добавлен ни один. Каждая клетка посещается один раз: time `O(rows·cols)`. Result занимает `O(rows·cols)`, auxiliary state `O(1)`.

### Edge cases

- `1 × 1`, `1 × n`, `m × 1`;
- нечётные размеры с центральной строкой/колонкой;
- пустая матрица: официальный constraint исключает её, production-функция должна проверить до `matrix[0]`;
- ragged matrix: алгоритм четырёх границ предполагает прямоугольник.

## Что надо уметь сказать без кода

1. Для каждой задачи назвать mapping координат или invariant границ.
2. Отделять input constraints от production validation.
3. Считать output space отдельно от auxiliary space.
4. Объяснить, почему каждая клетка обрабатывается ровно один раз или почему swap не повторяется.
5. До запуска проверить `1×1`, одну строку, один столбец, чётный и нечётный квадрат.
