# Структуры: 6 задач курса

## Урок 38. Value и pointer receivers

![[90 Вложения/CurseHunter/6609/Кадры/038.jpg]]

### Задача курса

- Method с value receiver получает копию struct, поэтому изменение scalar field не меняет original.
- Method с pointer receiver работает с исходным object.
- Но value receiver не означает deep immutability: slice, map, pointer или interface field в копии могут ссылаться на те же данные. Это подробно разобрано в [[60 Go/Pointer и value receivers, method sets|заметке о receivers и method sets]].

Method value фиксирует receiver при создании. В примере после `data.value = 200` два method values value-receiver возвращают `100`, а два method values pointer-receiver — `200`: вызов через pointer автоматически разыменовывается, но receiver всё равно захватывается по правилам объявленного method.

Вызов method на nil pointer receiver синтаксически допустим. Он panic только при dereference; method может сначала проверить `receiver == nil`. Вложенный pointer field объясняет ещё одну ловушку: value receiver копирует внешний struct, но mutation объекта через copied pointer видна original.

## Урок 39. Ambiguous selectors

![[90 Вложения/CurseHunter/6609/Кадры/039.jpg]]

Если два embedded fields на одной глубине продвигают `values` или `Print`, selector `base.values`/`base.Print()` ambiguous и код не компилируется. Решение — квалифицировать путь: `base.Derived1.values`, `base.Derived2.Print()` либо убрать конфликт из API.

## Урок 40. Alignment и padding

![[90 Вложения/CurseHunter/6609/Кадры/040.jpg]]

На показанной architecture:

```go
struct { bool; int32; bool } // 12 bytes
struct { int32; bool; bool } // 8 bytes
```

Порядок fields меняет padding, потому что address каждого field выравнивается под его alignment, а размер struct округляется под максимальное alignment. Но конкретные числа — implementation/architecture detail; API и serialization не должны полагаться на них. `[1023]byte` имеет alignment 1, поэтому `bool + array + bool` занимает `1025` bytes в примере.

## Урок 41. Embedding

![[90 Вложения/CurseHunter/6609/Кадры/041.jpg]]

Embedding — composition с promoted selectors, а не inheritance. Внешний `Woman.Intro` shadow promoted `Person.Intro`; квалифицированный `woman.Person.Intro()` остаётся доступен. Promotion влияет на method set по правилам value/pointer embedding, разобранным в [[60 Go/Embedding и composition|заметке об embedding]].

## Урок 42. Сравнение struct

![[90 Вложения/CurseHunter/6609/Кадры/042.jpg]]

Struct comparable, если comparable каждый field. Сравнение идёт по source order и прекращается на первом неравном field. Поэтому размещение дешёвого часто различающегося discriminator перед огромным array ускоряет negative comparison. Это micro-optimization: сначала выбирают корректный model/layout, затем подтверждают profile и benchmark.

## Урок 43. Empty struct

![[90 Вложения/CurseHunter/6609/Кадры/043.jpg]]

`struct{}{}` имеет size zero и полезен как value без payload, например в set `map[T]struct{}` или signal channel. Указатель на empty struct всё равно имеет размер pointer.

Разные zero-size variables могут иметь одинаковый address, а могут нет; сравнение их pointers не должно кодировать identity. Final zero-size field может увеличить размер enclosing struct: реализация вправе дать ему distinct address, не выходящий за allocation. В показанном примере `struct{ empty; int64 }` занимает `8`, а `struct{ int64; empty }` — `16` bytes.
