# Context: теория и 4 задачи курса

## Урок 106. Что предоставляет `context.Context`

![[90 Вложения/CurseHunter/6609/Кадры/106.jpg]]

**Коррекция слайда:** standard `context.Context` — interface, а не struct. Его methods сообщают deadline, cancellation через `Done`, причину через `Err` и request-scoped value через `Value`. Implementations образуют immutable parent chain; `WithCancel`, `WithDeadline`, `WithTimeout` и `WithValue` возвращают child.

Context передают первым argument, не хранят в long-lived struct без особой причины и не используют как мешок optional parameters. Модель распространения — в [[60 Go/Context, deadlines и распространение отмены|заметке о context]].

## Урок 107. Nil context

![[90 Вложения/CurseHunter/6609/Кадры/107.jpg]]

Вызов `ctx.Err()` при nil interface panic. Даже когда caller ещё не знает подходящий context, передают `context.TODO()`, а root программы создаёт `Background()`. API, принимающий context, не должен молча трактовать nil как Background: это скрывает bug и теряет cancellation.

## Урок 108. Parent и child

![[90 Вложения/CurseHunter/6609/Кадры/108.jpg]]

Child deadline не может эффективно продлить более ранний parent deadline. Parent timeout 2 секунды отменит child с requested 10 секунд примерно через 2. Cancel child не отменяет parent и siblings. Cancel parent распространяется вниз по tree.

Всегда вызывают returned `cancel`, даже если deadline скоро истечёт: это освобождает timer/reference раньше и делает ownership явным.

## Урок 109. `WithValue`

![[90 Вложения/CurseHunter/6609/Кадры/109.jpg]]

- Child с тем же comparable key shadow parent value, parent не меняется.
- Cancellation child наследует values parent.
- Plain string keys сталкиваются между packages. Используют unexported named key type; два named types с одинаковым underlying string различаются.
- Values предназначены для request-scoped metadata, проходящей через API/process boundary: trace ID, auth principal. Config и обязательные dependencies передают обычными arguments.

## Урок 110. Таймаут, который не отменяет работу

![[90 Вложения/CurseHunter/6609/Кадры/110.jpg]]

Course wrapper запускает `Query(query string)` в goroutine и возвращает `ctx.Err()` через 3 секунды. Buffered result channel не даёт завершившейся позже goroutine навсегда зависнуть на send, но underlying query всё равно продолжает расходовать connection/CPU.

Настоящая cancellation должна пройти до blocking operation:

```go
func Query(ctx context.Context, query string) (string, error)
```

Database/HTTP call обязан принимать context, а implementation — наблюдать его. Timeout ограничивает ожидание caller только тогда, когда dependency не поддерживает cancel; это отдельный resource-leak risk, который нужно ограничивать pool-ом и deadline на нижнем уровне.
