# Backend — от основ к архитектуре

Единый маршрут задаёт порядок освоения материала, а не уровни сложности. Каждая тема начинается с необходимых основ и доводится до Senior+-глубины: причинного механизма, инвариантов, failure modes, trade-offs и практических последствий.

## Порядок прохождения

1. **Алгоритмы, ОС и сети.** Начните с [[10 Основы CS/Карта — Основы CS|карты основ CS]]: структуры данных, сложность, процессы, память, ввод-вывод и сетевой стек.
2. **HTTP, API и безопасность.** Перейдите к [[20 Бэкенд/Карта — Бэкенд|карте бэкенда]]: жизненный цикл запроса, API-контракты, идемпотентность, authentication и authorization.
3. **Данные и кэширование.** Разберите [[30 Данные/Карта — Данные|карту данных]]: транзакции, индексы, изоляцию, модели хранения, репликацию и кэши.
4. **Координация и messaging.** Используйте [[40 Распределённые системы/Карта — Распределённые системы|карту распределённых систем]] для частичных отказов, согласованности, consensus и гарантий доставки.
5. **Reliability, performance и operations.** По [[70 Практические кейсы/Карта — Reliability, Performance и Operations|карте Reliability, Performance и Operations]] изучите SLO и error budgets, latency и saturation, observability, безопасные deployment, incident response и поиск bottleneck.
6. **Low-Level Design.** По [[50 Проектирование систем/Карта — Low-Level Design|карте LLD/OOD]] пройдите от предметных инвариантов и публичного API до конкурентной безопасности, тестируемости и реализации локальных компонентов.
7. **High-Level System Design.** По [[50 Проектирование систем/Карта — Проектирование систем|карте проектирования систем]] пройдите от требований и оценки нагрузки до границ сервисов, владения данными и архитектурных trade-offs.
8. **Эволюция систем.** Разберите multi-region, disaster recovery, миграции без простоя и изменение архитектуры под растущие ограничения.
9. **Практика и обратная связь.** Закрепляйте материал через [[70 Практические кейсы/Карта — Практические кейсы|практические кейсы]] и повторяйте опорные модели по [[80 Шпаргалки/Карта — Шпаргалки|шпаргалкам]].

## Параллельная подготовка опыта

По [[70 Практические кейсы/Карта — Behavioral, Resume и Project Deep Dive|карте Behavioral, Resume и Project Deep Dive]] связывайте утверждения resume с проверяемыми историями: личными решениями, альтернативами, метриками, ошибками и эксплуатационной ответственностью. Одни и те же факты можно использовать для разных behavioral-вопросов, если у каждого ответа свой тезис и причинная цепочка.

## Параллельное направление Go

Проходите [[60 Go/Карта — Go|карту Go]] параллельно этапам про бэкенд, распределённые системы и эксплуатацию: язык и runtime должны связываться с теми же системными механизмами и production-последствиями.

## Сквозная инженерная практика

Используйте [[20 Бэкенд/Карта — Testing, Debugging и Code Quality|карту Testing, Debugging и Code Quality]] на всём маршруте: проверяйте локальные правила и границы интеграции до релиза, диагностируйте деградации по наблюдаемым данным и оценивайте качество кода через стоимость безопасного изменения.
