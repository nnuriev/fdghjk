# Кратчайшие пути

## TL;DR

Вопрос заметки: как выбрать single-source shortest-path algorithm по структуре graph и весам рёбер, не применив доказательство за пределами его предусловий?

Выбор определяется не размером графа в первую очередь, а свойствами weights:

| Условия | Алгоритм | Типичная граница на adjacency lists |
| --- | --- | --- |
| все рёбра имеют unit/equal positive weight / graph unweighted | BFS | `Θ(V+E)` |
| graph — DAG, веса любые | topological relaxation | `Θ(V+E)` |
| все веса неотрицательны | Dijkstra с binary heap | `O((V+E) log V)` |
| есть отрицательные рёбра | Bellman–Ford | `O(VE)` |

Reachable negative cycle означает, что для вершин, достижимых из этого цикла, конечного shortest distance нет: цикл можно повторять и уменьшать стоимость без границы. Dijkstra при отрицательном edge применять нельзя; корректность его «окончательного» выбора опирается на неотрицательность всех дальнейших прибавлений.

## Ментальная модель

`dist[v]` — верхняя граница на истинную стоимость пути из source в `v`. Изначально известен только пустой путь в source: `dist[s]=0`, остальные равны infinity.

Relaxation ребра `u→v` веса `w` проверяет путь, который заканчивается этим ребром:

```text
если dist[u] + w < dist[v],
то dist[v] = dist[u] + w и parent[v] = u
```

Relaxation никогда не делает известный путь хуже. Алгоритмы отличаются порядком, в котором разрешают эти локальные улучшения, и моментом, когда считают distance окончательным:

- BFS упорядочивает по числу рёбер;
- DAG order обрабатывает vertex после всех predecessors;
- Dijkstra всегда выбирает минимальную tentative distance;
- Bellman–Ford повторяет все edges, пока пути с растущим числом рёбер не будут покрыты.

## Как устроено

### BFS и DAG

При одинаковом весе путь минимальной стоимости одновременно имеет минимальное число рёбер, поэтому работает layer invariant из [[10 Основы CS/BFS и DFS|BFS]]. Если weights равны константе `c>0`, BFS-distance умножается на `c`.

В DAG любой path следует topological order. Когда обрабатывается `u`, все пути в `u` уже прошли через ранее обработанных predecessors, поэтому `dist[u]` окончательна и outgoing edges достаточно relaxed один раз. Negative edges не мешают: вернуться и позже улучшить predecessor невозможно, потому что cycle отсутствует.

### Dijkstra

Dijkstra хранит unsettled vertices по tentative distance. Выбирается `u` с минимальным `dist`. Предположим, что более короткий путь в `u` существует, и возьмём на нём первую unsettled вершину `x`; её predecessor уже settled, поэтому relaxation установил `dist[x]` не больше стоимости prefix до `x`. Из выбора минимума `dist[u] ≤ dist[x]`, а nonnegative suffix не может сделать полный путь дешевле этого prefix. Получается противоречие, значит `dist[u]` уже оптимальна.

Отрицательное ребро ломает последнюю фразу: поздний путь может стать дешевле после прибавления отрицательного веса. Binary heap implementations часто добавляют новую пару `(distance, vertex)` вместо decrease-key; при pop stale entry пропускают, если записанная distance не равна текущей. Это сохраняет корректность, но число heap entries может достигать `O(E)`, а общая граница принимает вид `O((V+E) log E)`. Для simple graph она асимптотически совпадает с обычной записью `O((V+E) log V)`.

### Bellman–Ford

После `k` полных проходов по всем edges `dist[v]` не больше стоимости shortest walk из source в `v`, использующего не более `k` рёбер; induction следует из последнего ребра такого walk. Любой простой path содержит не больше `V-1` edges. Поэтому `V-1` проходов достаточно, если reachable negative cycle не участвует.

Если ещё один проход улучшает distance, существует улучшающий walk как минимум с `V` edges. Он повторяет vertex; повторённый фрагмент имеет отрицательный total weight, иначе удаление цикла не ухудшило бы walk. Проверять нужно только edges, чьи sources достижимы: negative cycle в отдельной компоненте не влияет на source.

### Арифметика и witness

Infinity нельзя представлять значением, которое затем без проверки складывают с `w`: `INF+w` может overflow. Сначала проверяют достижимость `u`, затем выполняют addition с типом и границами, соответствующими contract input. Общая политика overflow разобрана в [[10 Основы CS/Edge cases, невалидный ввод и overflow|заметке о границах и overflow]].

`parent` восстанавливает path только для конечной distance без влияния negative cycle. Для unreachable vertex parent отсутствует. Если tie-breaking важен, правило выбора равных путей должно входить в контракт, иначе разные корректные реализации вернут разные witnesses.

## Пример или трассировка

Directed weighted graph:

```text
s→a (4), s→b (5), a→b (-2), b→t (3), a→t (4)
```

Есть negative edge, но graph acyclic и negative cycle отсутствует. Возьмём edge order как в строке и выполним Bellman–Ford.

| Состояние | `d(s)` | `d(a)` | `d(b)` | `d(t)` | Объяснение |
| --- | ---: | ---: | ---: | ---: | --- |
| начало | 0 | ∞ | ∞ | ∞ | известен только source |
| `s→a` | 0 | 4 | ∞ | ∞ | путь `s→a` |
| `s→b` | 0 | 4 | 5 | ∞ | direct path в b |
| `a→b` | 0 | 4 | 2 | ∞ | `4 + (-2) = 2` улучшает b |
| `b→t` | 0 | 4 | 2 | 5 | `2 + 3 = 5` |
| `a→t` | 0 | 4 | 2 | 5 | кандидат `8` хуже |

Второй полный проход ничего не меняет. Итоговый shortest path в `t`: `s→a→b→t`, стоимость `5`.

Добавим `t→a (-10)`. Цикл `a→b→t→a` весит `-2+3-10=-9`. Каждый новый обход цикла уменьшает distances ещё на `9`; extra relaxation после `V-1` проходов обнаружит reachable negative cycle. Формулировка «shortest path равен очень большому отрицательному числу» неверна: минимума нет.

## Trade-offs

BFS и DAG relaxation линейны, но требуют сильной структуры. Dijkstra обычно быстрее Bellman–Ford, зато его proof целиком рушится от одного отрицательного edge. Bellman–Ford принимает negative weights и обнаруживает reachable negative cycles, платя `O(VE)`.

Если graph dense, простой Dijkstra с array и `O(V²)` иногда практичнее heap из-за констант и locality. Для weights `0/1` deque-based 0–1 BFS достигает `O(V+E)`: zero-weight relaxation идёт в начало deque, unit-weight — в конец. Это отдельный invariant, а не обычный BFS.

All-pairs question меняет выбор: repeated single-source algorithms и `O(V³)` dynamic programming решают разные density/weight regimes. Не стоит добавлять all-pairs machinery, если требуется один source-target path.

## Типичные ошибки

- **«Dijkstra работает, если negative edges редки» → settled vertex позднее получает более дешёвый путь → proof использовал неотрицательность каждого оставшегося edge → выбрать DAG relaxation или Bellman–Ford.**
- **«BFS минимизирует сумму weights» → путь с меньшим числом рёбер оказывается дороже → queue упорядочивает layers, а не cost → применять BFS только при одинаковом весе.**
- **«Negative cycle портит все distances» → отклоняется корректный путь в другой компоненте → влияет лишь cycle, достижимый из source, и vertices, достижимые из него → анализировать reachability влияния.**
- **«`V-1` relaxation одного edge order всегда достаточно за один проход» → distances не успевают пройти против порядка edges → один проход покрывает ещё одно ребро path в худшем случае → повторить полные проходы либо использовать topological order в DAG.**
- **«Infinity можно складывать как обычное число» → overflow превращает unreachable в дешёвый путь → sentinel вышел за арифметический контракт → проверять достижимость до addition и границы суммы.**
- **«Heap entry всегда актуален» → vertex обрабатывается многократно с устаревшей distance → push-without-decrease-key оставляет старые пары → skip stale entries.**

## Когда применять

1. Зафиксируйте directed/undirected graph, source(s), target и допустимый диапазон weights.
2. Проверьте специальные структуры: unit weights, `0/1`, DAG.
3. При общих nonnegative weights берите Dijkstra; при negative edges — Bellman–Ford либо DAG relaxation.
4. Отдельно определите semantics unreachable vertex, negative-cycle influence, overflow и восстановления path.
