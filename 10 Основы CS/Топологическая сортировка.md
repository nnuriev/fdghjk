# Топологическая сортировка

## TL;DR

Вопрос заметки: как построить порядок зависимостей и одновременно доказать, что такой порядок вообще существует?

Topological order directed graph — перестановка vertices, в которой для каждого edge `u→v` вершина `u` стоит раньше `v`. Такой порядок существует тогда и только тогда, когда graph — directed acyclic graph (DAG).

Алгоритм Kahn последовательно удаляет vertices с текущей indegree `0`. Если обработано меньше `V` вершин, оставшийся подграф содержит directed cycle. DFS даёт альтернативу: reverse finishing order корректен, только если во время обхода не найден back edge в active vertex.

Оба варианта работают за `Θ(V+E)` на adjacency lists. Конкретный порядок не обязан быть единственным; если нужен детерминированный лексикографически минимальный результат, zero-indegree set хранят в priority queue и платят логарифмический множитель.

## Ментальная модель

Indegree — число ещё не выполненных непосредственных prerequisites. Vertex с indegree `0` можно поставить следующим: ни одно оставшееся ребро не требует поместить другую оставшуюся вершину раньше неё.

Удаление такой вершины «выполняет» её и снимает по одному prerequisite с каждого outgoing neighbor. Получается invariant Kahn:

```text
indegree[v] = число входящих рёбер в v из ещё не обработанных вершин
```

Cycle блокирует процесс. У каждой вершины цикла остаётся predecessor в том же цикле, поэтому ни одна не получает indegree `0`. В DAG блокировки быть не может: если в конечном ненулевом подграфе у каждой вершины есть predecessor, переход по predecessors рано или поздно повторит vertex и образует cycle.

## Как устроено

### Почему порядок существует только для DAG

Если topological order существует, directed cycle `v₀→v₁→…→v₀` потребовал бы `pos(v₀)<pos(v₁)<…<pos(v₀)`, что невозможно. Значит graph с cycle не имеет порядка.

Обратная сторона конструктивна. В любом конечном DAG есть vertex с indegree `0`; иначе бесконечное следование по incoming edges в конечном множестве vertices повторило бы вершину. Удаляем zero-indegree vertex. Оставшийся graph тоже DAG, поэтому induction строит весь порядок. Это и есть доказательство Kahn.

### Kahn

Сначала один раз считают indegree всех vertices и помещают нулевые в queue. На каждом шаге:

1. извлекают одну zero-indegree vertex и добавляют в order;
2. для каждого outgoing edge уменьшают indegree neighbor;
3. neighbor, достигший нуля, помещают в queue.

Каждая vertex входит в queue один раз, каждый edge уменьшает один счётчик один раз. Время `Θ(V+E)`, state для indegrees, queue и результата занимает `Θ(V)`.

Если после опустошения queue длина order меньше `V`, необработанные vertices не обязательно все принадлежат cycle: часть может лишь зависеть от него. Гарантия точнее: в оставшемся подграфе есть хотя бы один cycle, а полного topological order нет.

### DFS-вариант

DFS добавляет vertex в результат при выходе, после всех descendants; reverse finishing order ставит источник раньше зависимых вершин. Но directed back edge `u→v`, где `v` находится в active recursion stack, доказывает cycle. Без этой проверки reverse postorder циклического graph выдаст список, который выглядит полным, но нарушает хотя бы одно ребро.

White/gray/black states из [[10 Основы CS/BFS и DFS|DFS]] отделяют unseen, active и finished vertices. Kahn обычно проще, когда нужен явный cycle/no-cycle outcome или обработка indegree естественно соответствует предметным prerequisites. DFS удобен, если traversal уже нужен для другой структурной задачи.

### Неединственность и стабильность

Одновременно доступные zero-indegree vertices независимы в текущем partial order. FIFO queue, stack и heap выберут разные корректные linear extensions. Если downstream tests сравнивают один конкретный массив, tie-breaking должен входить в контракт. Min-heap даёт лексикографически минимальный доступный выбор за `O(E + V log V)`: edges по-прежнему меняют счётчики за constant time, а каждая vertex один раз входит в heap и выходит из него.

## Пример или трассировка

Пусть edges задают prerequisites:

```text
A→C, B→C, B→D, C→E, D→E
```

Начальные indegrees: `A=0`, `B=0`, `C=2`, `D=1`, `E=2`. FIFO queue стартует как `[A,B]`.

| Шаг | Извлечён | Изменённые indegrees | Queue после шага | Order |
| ---: | --- | --- | --- | --- |
| 1 | A | `C: 2→1` | `[B]` | `[A]` |
| 2 | B | `C: 1→0`, `D: 1→0` | `[C,D]` | `[A,B]` |
| 3 | C | `E: 2→1` | `[D]` | `[A,B,C]` |
| 4 | D | `E: 1→0` | `[E]` | `[A,B,C,D]` |
| 5 | E | — | `[]` | `[A,B,C,D,E]` |

Каждое ребро направлено слева направо в результате. Порядок `[B,A,D,C,E]` тоже корректен, что подтверждает неединственность.

Добавим edge `E→B`. Тогда `B` перестаёт быть начальной zero-indegree vertex. После обработки `A` очередь пустеет, а четыре вершины остаются. `processed < V` обнаруживает цикл `B→C→E→B` (также есть `B→D→E→B`).

## Trade-offs

Kahn хранит indegrees и естественно поддерживает динамику «готовых» задач. Он сразу показывает stalled state, но сам по себе не восстанавливает конкретный cycle; для witness нужен отдельный обход оставшегося subgraph.

DFS требует colors и call/explicit stack. Зато finishing order уже доступен для дальнейших graph algorithms, а back edge позволяет восстановить cycle через parent chain. Глубокий path даёт `Θ(V)` stack depth.

Topological order решает precedence, но не scheduling с durations, ограниченным числом workers и resource conflicts. DAG shortest/longest paths используют порядок для relaxation, однако это соседняя задача, а не свойство самого списка.

## Типичные ошибки

- **«Любой directed graph можно топологически отсортировать» → алгоритм возвращает неполный список или нарушенное edge constraint → cycle создаёт противоречивый строгий порядок → проверить `processed == V` либо back edges.**
- **«Vertex с исходной indegree `0` достаточно собрать один раз» → новые готовые vertices никогда не обрабатываются → indegree меняется после удаления prerequisites → enqueue neighbor именно при переходе счётчика к нулю.**
- **«Reverse DFS finish всегда topological» → циклический graph получает правдоподобный, но неверный список → не проверен active-stack back edge → использовать colors и отклонить graph при cycle.**
- **«Topological order единственный» → тест ломается на другом порядке queue/map iteration → partial order допускает несколько linear extensions → проверять edge constraints либо определить tie-breaking.**
- **«Все необработанные vertices лежат на cycle» → диагностика помечает downstream dependents как членов цикла → Kahn доказывает наличие цикла в остатке, а не membership каждой вершины → отдельно восстановить strongly connected/cycle witness.**

## Когда применять

- Build dependencies, course prerequisites, pipeline stages и schema migrations, если relation направлена и обязана быть acyclic.
- DAG dynamic programming и relaxation, когда predecessor state должен быть готов раньше successor.
- Kahn — для explicit ready queue и проверки `processed == V`; DFS — когда нужны finishing times или cycle witness через traversal state.
- Если dependencies меняются во время работы, обычный одноразовый sort не заменяет incremental cycle/order maintenance.
