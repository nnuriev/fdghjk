# Битовые операции и базовая математика

## TL;DR

Вопрос заметки: как использовать bit manipulation, gcd и modular arithmetic, не опираясь на неверную модель integer arithmetic в Go?

Для фиксированного `n`-битного unsigned word арифметика идёт modulo `2^n`; bits удобно рассматривать как set позиций. `&`, `|`, `^` и `&^` реализуют intersection, union, symmetric difference и clear. Shift двигает позиции, но его тип и переполнение определяются типом левого operand.

В Go 1.26 unsigned operations вычисляются modulo `2^n`. Signed overflow для typed integers разрешён и детерминирован, panic не происходит. Runtime division by zero и negative shift count вызывают panic. `int`/`uint` имеют 32 или 64 bits в зависимости от реализации. Narrowing conversion может молча отбросить старшие bits. Поэтому bit tricks лучше писать над `uintN`, а arithmetic bounds и conversions проверять отдельно.

Из базовой математики чаще всего нужны Euclidean gcd, remainder semantics, нормализация residue и защита промежуточных операций от overflow. Выражение `(a*b)%m` неверно как overflow-safe modular multiplication: произведение может переполниться до взятия remainder.

## Ментальная модель

Unsigned value — кольцо из `2^n` состояний: после максимума идёт ноль. Bit mask — множество включённых позиций:

```text
x & mask   оставить позиции mask
x | mask   включить позиции mask
x ^ mask   переключить позиции mask
x &^ mask  очистить позиции mask
```

У signed value тот же конечный machine word, но сравнение и right shift интерпретируют старший bit как знак. Поэтому трюк, доказанный для nonnegative bit set, не стоит механически переносить на signed inputs.

Euclidean algorithm использует другой инвариант: общие делители `a` и `b` совпадают с общими делителями `b` и `a mod b`. Замена пары уменьшает второй nonnegative operand, пока remainder не станет нулём. Modular arithmetic склеивает integers с одинаковым residue, но Go `%` возвращает remainder, знак которого следует dividend; canonical residue `[0,m)` нужно нормализовать явно.

## Как устроено

### Точная семантика Go 1.26

Language specification Go 1.26 фиксирует следующие границы:

- `uint8`, `uint16`, `uint32`, `uint64` и `uintptr` используют arithmetic modulo `2^n`;
- signed `+`, `-`, `*`, `/` и `<<` могут overflow; результат существует и определяется представлением и operands, compiler не вправе считать `x < x+1` всегда истинным;
- untyped constants имеют arbitrary precision, но assignment/conversion требует representability; typed runtime values уже ограничены шириной;
- integer division truncates toward zero, remainder удовлетворяет `x = q*y + r` и имеет знак dividend; runtime divisor `0` вызывает panic;
- особый signed case `minInt / -1` возвращает `minInt`, remainder `0` из-за overflow;
- shift count должен быть nonnegative integer; negative runtime count вызывает panic, верхнего предела count нет; результат имеет тип left operand;
- right shift signed integer arithmetic, unsigned integer logical;
- conversion между integer types сначала расширяет знак до неограниченной точности, затем обрезает до destination width; conversion не сообщает об overflow.

Эти правила проверены по specification language version go1.26 от 12 января 2026 года; текущий patch release на дату проверки — Go 1.26.5 от 7 июля 2026 года. Практические последствия conversions и bounds связаны с [[10 Основы CS/Edge cases, невалидный ввод и overflow|обработкой edge cases и overflow]].

### Маски и устойчивые идиомы

Для `x` типа `uint64` и позиции `k`:

- test: `(x >> k) & 1`;
- set: `x | (uint64(1) << k)`;
- clear: `x &^ (uint64(1) << k)`;
- toggle: `x ^ (uint64(1) << k)`.

Если `k` приходит как signed runtime input, сначала проверяют `k ≥ 0`; иначе shift panics. При `k` не меньше width typed left operand биты сдвигаются за границу, результат не «оборачивает» позиции.

`x != 0 && x&(x-1) == 0` проверяет, что unsigned `x` содержит ровно один set bit. Условие `x != 0` обязательно: при `x=0` unsigned subtraction wraps, а итоговое `0 & maxUint` тоже равняется нулю. Операция `x & (x-1)` очищает lowest set bit; повторение считает set bits за `O(popcount)`. Для стандартных fixed-width типов `math/bits` уже предоставляет `OnesCount`, `LeadingZeros`, rotations, additions with carry и wide multiplication.

### GCD и modular arithmetic

Для nonnegative `a,b` Euclid повторяет `(a,b) ← (b,a mod b)` до `b=0`; оставшийся `a` равен gcd. Extended Euclid дополнительно поддерживает coefficients `x,y`, для которых `ax+by=gcd(a,b)`. Modular inverse `a⁻¹ mod m` существует тогда и только тогда, когда `gcd(a,m)=1`.

При `m>0` Go expression `r = x % m` даёт `-(m-1) < r < m`. Для canonical residue:

```text
r = x % m
если r < 0, r += m
```

Одного условного addition достаточно. Оно безопаснее шаблона `(x%m + m)%m` ещё и тем, что не выполняет лишнюю операцию; после remainder отрицательное `r` строго больше `-m`, поэтому `r+m` лежит в `[1,m-1]`.

Modular addition и multiplication требуют контроля промежуточного значения. Если operands близки к границе `uint64`, обычное `a*b` уже вычислено modulo `2^64` до `%m`. Нужны доказанные bounds, checked decomposition, `math/bits.Mul64` с high/low words или arbitrary-precision `math/big` в зависимости от контракта.

### Модель стоимости

Bit operation над fixed-width Go integer считается `O(1)` в word-RAM model. Для arbitrary-precision integer стоимость зависит от числа machine words; её нельзя называть constant относительно bit length. Euclid выполняет `O(log min(a,b))` divisions в стандартной оценке по величине operands.

Overflow-safe midpoint при `0 ≤ lo ≤ hi` записывают как `lo + (hi-lo)/2`; прямое `(lo+hi)/2` может переполниться. Для сравнения `x*x ≤ n` при positive integers иногда используют `x ≤ n/x`, чтобы не вычислять потенциально переполняющий квадрат.

## Пример или трассировка

### Очистка младшего set bit

Возьмём unsigned `x=40`:

```text
x     = 101000₂
x - 1 = 100111₂
x & (x-1) = 100000₂ = 32
```

Вычитание меняет lowest set bit на `0`, а все младшие нули — на `1`; AND удаляет добавленные единицы и оставляет старшие bits. Повтор: `32 & 31 = 0`. Значит у `40` было два set bits.

### Euclid

Для `gcd(252,105)`:

| Шаг | Деление | Новая пара |
| ---: | --- | --- |
| 1 | `252 = 2·105 + 42` | `(105,42)` |
| 2 | `105 = 2·42 + 21` | `(42,21)` |
| 3 | `42 = 2·21 + 0` | `(21,0)` |

Итог `gcd=21`. На каждом шаге множество общих делителей сохраняется, а второй operand уменьшается; это одновременно correctness invariant и termination argument.

## Trade-offs

Bit mask компактен и даёт быстрые set operations, когда universe мал и позиции стабильны. При большом или sparse universe обычный set/map яснее и не привязан к ширине word. Маска также ограничивает число элементов и требует явного mapping «entity → bit».

Ручной bit trick может сократить пару операций, но стандартный `math/bits` лучше документирует намерение и покрывает architecture-specific implementation. При этом package не отменяет preconditions: например, division helpers имеют собственные требования и panic cases.

Primitive integers быстрее и проще arbitrary precision, пока доказано отсутствие нежелательного overflow. `math/big` сохраняет математический результат для больших значений, но меняет complexity, allocations и API. Checked arithmetic через comparisons или `math/bits` занимает промежуточное место.

## Типичные ошибки

- **«Signed overflow в Go вызывает panic» → код молча получает wrapped/deterministic machine result → specification разрешает overflow → проверять границы до операции, если overflow нарушает domain contract.**
- **«`int` всегда 64-bit» → решение ломается на 32-bit target или при больших constraints → ширина implementation-specific: 32 или 64 → использовать подходящий fixed-width type и проверять conversions.**
- **«`x&(x-1)==0` распознаёт степень двойки» → ноль ошибочно проходит проверку → unsigned `0-1` wraps, а AND остаётся нулём → добавить `x != 0`.**
- **«`(a*b)%m` защищено modulus» → промежуточное multiplication overflow происходит раньше → порядок evaluation уже потерял high bits → доказать bounds или использовать wide/checked multiplication.**
- **«`x%m` всегда лежит в `[0,m)`» → negative input даёт negative remainder → Go remainder следует знаку dividend → при `m>0` один раз добавить `m`, если `r<0`.**
- **«Shift count автоматически ограничится шириной» → отрицательный count panics, большой count обнуляет/заполняет результат вместо rotation → shift и rotation имеют разные semantics → валидировать count или использовать `math/bits.RotateLeft*`.**

## Когда применять

- Bit masks: subsets небольшого universe, flags, parity, compact visited states и powers-of-two constraints.
- `math/bits`: popcount, leading/trailing zeros, rotations, carry/borrow и 128-bit intermediate из двух `uint64` words.
- Euclid/extended Euclid: gcd, coprimality, fraction reduction, modular inverse при `gcd=1`.
- До arithmetic назовите width, signedness, допустимость negative values, overflow policy и modulus range. Без этого bit trick не имеет полного контракта.
